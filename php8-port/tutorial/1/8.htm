7. СПИСКИ ДИСКОВ, ПАПОК И ФАЙЛОВ

Всякий пользователь персонального компьютера знает, что эффективность его работы с приложениями зависит от организации пользовательского интерфейса. Организация быстрого и удобного поиска необходимой информации в памяти компьютера, реализация разноообразных потребностей доступа к данным является неотъемлемой частью работ при создании практически любого приложения, поэтому программирование той части приложений, которая связана с перемещением по дискам, папкам и файлам, является важной составной частью работы программиста. Во многом эффективность такой работы зависит от возможностей среды разработки приложений.

В главе 3 при программировании диалогов использованы стандартные диалоговые компоненты SaveDialog, OpenDialog и др., которые позволяют выполнять ряд таких действий. Однако организация этих диалогов такова, что зачастую доступ к ряду необходимых операций ограничен, что делает их недоступными при работе с дисковой памятью.

Среда Delphi поддерживает четыре визуальных компонента, которые дают возможность организовать работу с дисками, папками и файлами в соответствии с потребностями программиста.

П р и м е ч а н и е. В среде Delphi сохранился устаревший к настоящему времении термин директория (directory), унаследованный из терминологии DOS. В операционной системе Windows принято использовать вместо него термин папка (folder), который является синонимом директории. Далее будем использовать именно этот термин, подразумевая абсолютную идентичность этих терминов.

Компоненты
Список папок (компонент DirectoryListBox, закладка Win3.1). Отображает текущую папку и позволяет пользователю переходить от одной папки к другой папке.
Комбинированный список дисков (компонент DriveComboBox, закладка Win3.1). Отображает список всех дисковых накопителей, включая жесткие диски, дискеты, компакт-диски и др.
Список файлов (компонент FileListBox, закладка Win3.1). Отображает файлы в текущей папке. Отображает все файлы, либо файлы, которые соответствуют выбранному шаблону, например, *.pas, *.exe и т д.
Комбинированный список фильтров (компонент FilterComboBox, закладка Win3.1). Перечисляет фильтры, предназначенные для отбора файлов определенного типа.

Приложение для просмотра списков файлов, папок и дисков

Четыре перечисленных компонента Delphi – это средства, которые позволяют организовать разнообразные процедуры диалоговой работы с дисками, папками и файлами. Такие диалоги могут полностью повторять или представлять модернизированные версии уже известных нам диалогов OpenDialog, SaveDialog или других, а также представлять другие диалоговые окна как по форме, так и по содержанию, с новыми возможностями и функциями в зависимости от потребности и назначения.

Для ознакомления с возможносяти этих компонентов разработаем простое приложение, которое позволяет просмотреть содержимое имеющихся дисков.

Войдите в Delphi. Если оно содержит приложение, закройте его и создайте новое приложение.

Дайте форме Form1 имя fMain, свойству Caption присвойте название Просмотор дисков и папок. Сохраните модуль под именем main.pas в новой папке FolderApp и приложение – под именем folders.dpr в той же папке.

Внешний вид окна проектируемого приложения показан на рис. 7.1.

Рис. 7.1. Вид формы программы FolderApp

Мысленно рабочую площадь формы можно разделить на две вертикальные зоны. В левой зоне, располагая сверху вниз, положите метку Label (закладка Standard; дайте ей имя FileLabel, наименование – Имя файла:), строку ввода Edit (закладка Standard), компонент FileListBox (закладка Win 3.1), еще одну метку Label (наименование – Файлы заданного типа:) и компонент FilterComboBox (закладка Win 3.1).

В правой зоне последовательно положите одну под другой три метки Label. Первой метке дайте наименование Папки:, второй – имя FolderLabel, третей – имя FolderLabel2. Под ними положите компонент DirectoryListBox (закладка Win 3.1), метку Label. Метке дайте наименование Диски:. В нижней части положите компонент DriveComboBox (закладка Win 3.1) и кнопку BitBtn (закладка Additional). Измените значение свойства Kind кнопки на bkClose.

Разместите и измените размеры компонентов приблизительно так, как показано на рис. 7.1.

Далее создайте два обработчика событий: OnCreate – для формы и OnChange – для компонента DirectoryListBox. Текст модуля main, включая процедуры для обработчиков, приведены в листинге 7.1.

Листинг 7.1. Модуль main проекта FolderApp

unit main;

interface

uses

Windows, Messages, SysUtils, Classes, Graphics, Controls,
Forms, Dialogs, StdCtrls, Buttons, FileCtrl;

type

TfMain = class(TForm)
bbClose: TBitBtn;
FileListBox1: TFileListBox;
DirectoryListBox1: TDirectoryListBox;
FilterComboBox1: TFilterComboBox;
DriveComboBox1: TDriveComboBox;
Label1: TLabel;
Label2: TLabel;
Edit1: TEdit;Label3: TLabel;
FolderLabel: TLabel;
Label5: TLabel;
FolderLabel2: TLabel;
procedure FormCreate(Sender: TObject);
procedure DirectoryListBox1Change(Sender: TObject);
private
{ Private declarations }
public

{ Public declarations }

end;

var

fMain: TfMain;

implementation

{$R *.DFM}

procedure TfMain.FormCreate(Sender: TObject);
begin
DirectoryListBox1.FileList:=FileListBox1;
DirectoryListBox1.DirLabel:=FolderLabel;
FolderLabel2.Caption:= DirectoryListBox1.Directory;
DriveComboBox1.DirList:=DirectoryListBox1;
FileListBox1.FileEdit:=Edit1;
FilterComboBox1.FileList:=FileListBox1;
end;

procedure TfMain.DirectoryListBox1Change(Sender: TObject);
begin
FolderLabel2.Caption:= DirectoryListBox1.Directory;
end;

end.

Событие OnCreate для формы fMain обрабатывается процедурой TfMain.FormCreate. Это событие происходит при создании формы, т. е. при организации и размещении ее в оперативной памяти. При этом последовательно исполняются шесть операторов, составляющих тело этой процедуры.

Компонент DirectoryListBox

При исполнении первого оператора

DirectoryListBox1.FileList:=FileListBox1;

приложению отдается указание о том, что списком файлов FileList текущей папки из списка папок DirectoryListBox1 (на форме список папок размещен справа) является список FileListBox1 (он – слева).

При исполнении второго оператора

DirectoryListBox1.DirLabel:=FolderLabel;

приложению отдается указание о том, что полное имя текущей папки, включая родительские папки и диск, следует отображать в наименовании метки FolderLabel.

Для отображения длинных маршрутов объект DirectoryListBox заменяет многоточием имена папок, расположенных около корневой папки. В результате получится относительно небольшой текст наименования метки, равный по ширине стандартному размеру этого компонента. Например, маршрут

C:\Program Files\Borland\Delphi4\Demos\Activex

компонент DirectoryListBox1 покажет в виде

C:\...\Borland\Delphi4\Demos\Activex

Для того чтобы отобразить полное имя папки без сокращений, можно использовать свойство Directory компонента DirectoryListBox. Так, третий оператор

FolderLabel2.Caption:= DirectoryListBox1.Directory;

присваивает значение текущей директории DirectoryListBox1.Directory наименованию метки FolderLabel2.

Обратите внимание, что такой же оператор помещен в тело обработчика OnChange события TfMain.DirectoryListBox1Change. Это событие происходит тогда, когда пользователь переходит к другой папке. Если бы обработчик не содержал такого оператора, то в наименовании метки остался бы тот маршрут, который был присвоен ему при создании формы, что при дальнейшем использовании метки могло бы вызвать серьезные неполадки в работе программы.

Компонент DirectoryListBox имеет свойство Drive, которое может быть использовано для определения литеры текущего диска. В следующем операторе символу z, имеющему тип Char, присваивается символ, соответствующий текущему диску:

z:= DirectoryListBox1.Drive;

Компонент DriveComboBox

При выполнении четвертого оператора события TfMain.FormCreate

DriveComboBox1.DirList:=DirectoryListBox1;

приложению отдается указание о том, при выборе текущего диска из комбинированного списка дисков DriveComboBox1 список папок DirList выбранного диска следует разместить в списке DirectoryListBox1.

С помощью свойства Items объекта DriveComboBox можно получить доступ к списку доступных дисков. Например, испльзуя следующий оператор, можно поместить этот список в список ListBox1

ListBox1.Items:= DriveComboBox1.Items;

Компонент FileListBox

При выполнении пятого оператора события TfMain.FormCreate

FileListBox1.FileEdit:= Edit1;

приложению отдается указание о том, что после выбора файла из списка FileListBox1 (список файлов расположен слева) щелчком мыши либо с помощью клавиш его имя FileEdit следует поместить в строку ввода Edit1. Такая организация взаимодействия списка со строкой ввода очень удобна для выбора имени файла. Она позволяет быстро выбрать уже существующий файл, либо ввести с клавиатуры имя нового файла. Часто при вводе нового имени подбирают близкое имя существующего файла, помещают его описанным способом в строку ввода и при минимуме изменений быстро получают требуемое имя для дальнейшего использования.

При активизации формы строке ввода присваивается начальное значение *.*, т. е. отображается фильтр, обозначающий имена всех файлов. Если необходимо, чтобы пользователь мог выбрать файл только из существующего списка, установите значение свойства строки ввода (в нашем примере – для компонента Edit1) ReadOnly (только чтение) равным True. Этим Вы сделаете невозможной коррекцию строки ввода.

В объекте FileListBox имя выбранного из списка файла является значением свойства FileName. Это имя является полным, т. е. содержит литеру диска, маршрут и собственно имя файла, например

C:\Program Files\Borland\Delphi4\Demos\Activex\argus.tmp

При необходимости его можно использовать для отделения имени, маршрута или диска. Для этого можно использовать файловые функции Delphi. С помощью следующего кода наименованием метки Label1 станет имя файла, метки Label2 – маршрут, метки Label3 – расширение файла, метки Label4 – литера диска с символом ":" (двоеточие).

With FileListBox1 do
begin
Label1.Caption:= ExtractFileName(FileName);
Label2.Caption:= ExtractFileExt(FileName);
Label3.Caption:= ExtractFilePath(FileName);
Label4.Caption:= ExtractFileDrive(FileName);
end;

Например, если

FileListBox1.FileName = 'C:\ProgramFiles\Borland\arg.tmp',

то действие этого кода идентично действию следующих операторов:

Label1.Caption:= 'arg.tmp';
Label2.Caption:= '.tmp';
Label3.Caption:= 'C:\Program Files\Borland\';
Label4.Caption:= 'C:';

Пользователи могут выбирать сразу несколько файлов, если свойство MultiSelect равно True. При этом строка ввода будет отображать имя последнего выбранного файла.

Компонент FilterComboBox

Наконец, при выполнении шестого (последнего) оператора события TfMain.FormCreate

FilterComboBox1.FileList:=FileListBox1;

приложению отдается указание о том, что отбор файлов в список FileListBox1 следует производить в соответствии с шаблоном, который выбран из списка шаблонов FileList комбинированного списка фильтров FilterComboBox1.

Для того чтобы внести в список шаблонов необходимые фильтры, щелкните по компоненту FilterComboBox1, затем в окне Object Inspector – по кнопке подстановки данных свойства Filter. При этом откроется окно редактора Filter editor, представленное на рис. 7.2.

В групповых фильтрах отделяйте шаблоны друг от друга символом ";" (точка с запятой). В таких фильтрах не должно быть пробелов. По окончании редактирования нажмите кнопку ОК.

Строки левого столбца будут отображаться в объекте FilterComboBox. С помощью шаблонов правого столбца в объект FileListBox будут отобраны только те файлы, которые удовлетворяют выбранному фильтру.

Для того чтобы отобрать файлы требуемого шаблона, щелкните по кнопке справа на объекте FilterComboBox, затем – по требуемому фильтру. Если в текущей папке списка DirectoryListBox есть файлы, соответствующие выбранному фильтру, то они будут показаны в списке FileListBox, иначе этот список будет пуст.
