6. ИСПОЛЬЗОВАНИЕ СПИСКОВ


Существует структура данных, которая рано или поздно всегда потребуется каждому программисту. Такой структурой является список. Это один из самых полезных и необходимых управляющих элементов Windows. С его помощью, а также с помощью его ближнего"родственика" – комбинированного списка, который снабжен также строкой ввода текста, можно создавать практически любые списки строк. Можно сортировать данные списка, а также использовать списки различных типов, например с множественным выбором или поиском определенных элементов.

Кроме того в Delphi поддерживается несколько родственных классов для создания таких списков, как TList, TStrings, TStringList. Все они составляют мощный и весьма полезный набор инструментов для создания интерфейсов пользователя.

Визуальные компоненты
Список (компонент ListBox, закладка Standard). Используется как стандартное управляющее средство Windows для создания списков строк, из которых пользователь может выбирать элементы, используя команды мыши или клавиатуры.
Комбинированный список (компонент ComboBox, закладка Standard). Стандартное средство Windows. Объединяет строку ввода со списком. Пользователи могут выбирать один из элементов списка, перечень которых при желании можно вызвать в раскрывшемся списке, либо вводить новые данные в строке ввода.

Компонент ListBox

Этот компонент обладает рядом интересных свойств, которые позволяют создавать списки различных стилей. Например, объекты ListBox обычно обрамлены черной рамкой, но Вы можете установить свойство BorderStyle (стиль обрамления) равным bsNone для получения списка без рамки. Список такого вида хорошо смотрится на объекте Bevel (фаска). Для того, чтобы отобразить список в несколько колонок, достаточно присвоить нужное значение свойству Columns (столбцы). По умолчанию оно равно нулю. Если необходим список с несколькими колонками, установите его равным 2 или больше.

Данные в список можно заносить по-разному. Первый способ состоит в том, что список может быть создан заранее при разработке программы. Для этого нужно щелкнуть по значению свойства Items, затем в открывшемся окне редактора String List Editor – набрать строки списка и закрыть редактор кнопкой ОК. При этом заготовленный список будет сохранен при компиляции приложения в загрузочном exe-модуле (это несколько увеличивает объем модуля) и при запуске программы список будет автоматически отображен в объекте.

Второй способ состоит в формировании списка в процессе работы программы. Пограммно список может быть сформирован при вызове соответствующего метода для свойства Items, которое является объектом класса TStrings. Всякий объект ListBox имеет объект Items для простого доступа к данным списка. Например, для того чтобы пользователь мог выполнить ввод новых строк, а затем добавиь их в список, воспользуйтесь следующим кодом:

Var
S: String;
begin
S:=InputBox('Программа тестирования', 'Введите строку','');
if Length(S)>0 then ListBox1.Items.Add(S);
end;

Здесь использована строковая переменная S, в которую будет помещено значение строки символов, введенной пользователем в окне ввода, создаваемого на экране вызовом функции InputBox. Оператор if при помощи функции Length проверит длину (количество символов) этой строки и, если длина окажется большей нуля, то содержимое строки S будет добавлено в список ListBox1. Обратите внимание, что вызывается метод Add для объекта Items, который принадлежит объекту-списку ListBox1. Сам ListBox1 метода Add не имеет. Отметим, что код ListBox1.Items.Add(S) можно записать иначе с помощью оператора with:

With ListBox1.Items do
Add(S);

Для получения данных из списка используйте Items как массив. Обратите внимание, что первая запись имеет нулевой номер. Например, оператором

S:= ListBox1.Items[0];

первая строка списка будет скопирована в строку S.

Можно эту же операцию выполнить иначе:

S:= ListBox1.Items.Strings[0];

т. к. в действительности массивом является объект Strings, принадлежащий объекту Items.

Для получения доступа ко всем строкам списка используйте свойство Count, которое указывает на количство записей в списке. Например, с помощью следующего кода, используя цикл, можно последовательно извлечь из списка все строки с выводом каждого из них в модальное окно ShowMessage

Var
S: String; i: Word;
Begin
With ListBox1.Items do
for i:= 0 to Count-1 do
begin
S:=Strings[i];
ShowMessage(S);
end;
end;

Можно заменить два оператора, расположенные в теле цикла, одним

ShowMessage(Strings[i]);

Список можно загружать прямо из файла. Например, если файл 'a. txt' состоит из строк символов, то его содержимое можно загрузить в список следующим оператором

ListBox1.Items.LoadFromFile('a.text');

Можно загружать списки из других списков. Например следующий оператор позволит к уже имеющемуся списку ListBox1 добавить все строки из списка ListBox2

ListBox1.Items.AddStrings(ListBox2);

Для того, чтобы поменять местами две строки списка следует использовать метод Exchange. Например, следущий оператор поменяет местами первую и шестую строки списка LB5

LB5.Items.Exchange(1,6);

Для удаления элемента списка можно использовать метод Delete. Например, следущий оператор удалит шестую строку списка LiBiL:

LiBi.Items.Delete(5);

Для очистки списка используйте метод Clear:

ListBox2.Items.Clear;

Приложение ListFont

Программа ListFont показывает, как в объект ListBox можно вставить перечень названий системных шрифтов, а затем отобразить какой-нибудь элемент списка как отдельную от списка строку.

На рис. 6.1 показано состояние формы при проектировании приложения. На ней размещен на фаске Bevel компонент ListBox с именем FontList, кнопка SpinEdit с именем seFontSize, справа – еще одна фаска, на которой размещена метка Label с именем FontLabel, и кнопка BitBtn с именем bbClose.

Создайте новое приложение и сконструирйте форму, как показано на рис 6.1.

После конструирования формы создайте три обработчика события:
для щелчка по строке списка (OnClick для FontListBox) –

procedure TListFont.FontListBoxClick(Sender: TObject);
для щелчка по кнопкам объекта seFontSize (событие OnChange ) –

procedure TListFont.seFontSizeChange(Sender: TObject);
для автоматической загрузки системных шрифтов в момент активизации формы (OnActivate для ListForm) –

procedure TListFont.FormActivate(Sender: TObject);

После запуска программы при активизации формы в список загружаются наименования всех имеющихся шрифтов. Это происходит при выполнении процедуры TListFont.FormActivate, точнее размещенного в ней оператора

FontListBox.Items:= Screen.Fonts;

Если щелкнуть по любой строке списка, то текст метки FontLabel (она расположена справа от списка) изменится на текст выбранной строки. При этом одновременно изменится шрифт, при помощи которого будет выведен этот текст: он будет иметь начертание в графике выбранного шрифта. Как видно из рис. 6.2, щелчок был произведен по строке Comic Sans MS. При этом справа надпись на метке повторяет выбранную надпись. Эта надпись изображена в графике выбранного шрифта. Сами строки списка имеют шрифт, установленный при конструировании экранной формы (в данном случае фонт MS Sans Serif).

Рис. 6.2. Вид формы работающей программы ListFont

Эти действия выполняются с помощью операторов

FontLabel.Font.Name:= FontListBox.Items[ItemIndex];

FontLabel.Caption := FontListBox.Items[ItemIndex];

При исполнении первого из этих операторов метка (FontLabel) получает для "своего" шрифта (Font) наименование (Name) из той строки, по которой произведен щелчок (индекс этой строки равен значению переменной ItemIndex). При исполнении второго оператора наименование (Caption) этой метки станет равным наименованию выбранной строки.

Щелчок по любой из кнопок объекта SpinEdit повлечет изменение размера шрифта метки FontLabel. Механизм изменения таков. В тот момент, когда пользователь щелкает по кнопке, происходит изменение значения (Value) в редактируемой строке SpiEdit, что в свою очередь вызывает событие OnChange (на изменение) и активизацию соответствующего обработчика события. При его обработке произойдет выполнение оператора

FontLabel.Font.Size:= seFontSize.Value;

т. е. замена у метки (FontLabel) размера (Size) шрифта (Font) на выбранное значение кнопки (seFontSize.Value).

Компонент ListBox весьма удобен для тех приложений, в которых необходимо организовать множественный выбор. Обычно такой выбор используют для группового удаления, копирования или переноса записей. Если необходимо установить режим множественного выбора, установите значение его свойства MultiSelect равным True. Это можно сделать как при проектировании приложения, так и программно. Например, оператор

ListBox1.MultiSelect:= true;

переведет объект ListBox1 в мультиселектный режим.

На выбор оказывает влияние значения свойства ExtendedSelect (расширенный выбор). Если оно установлено в False, то щелчок по строке списка сделает строку выбранной, если она еще не выбрана, и, наоборот, снимет выбор, если до щелчка она была уже выбрана. Выделенные строки окрашены в темно-синий цвет.

На практике однако используют значение ExtendedSelect = True, т. к. этот режим отличается большей гибкостью. Существует два метода выбора. Условно их можно назвать дискретным и непрерывным.

Дискретный метод выбора. Если нужно выделить отдельные записи, то следует нажать клавишу Ctrl и произвести щелчки по строкам, подлежащим выделению. Если нужно выделить целый массив строк, то следует нажать клавишу Shift, щелкнуть по первой записи, а затем по последнеей записи. При этом весь массив заключенных между ними строк, включая крайние строки, по которым были произведены щелчки, окажется выделенным.

Непрерывный метод выбора. Нажите клавишу Ctrl. Не отпуская ее, нажмите левую клавишу мыши и в таком состоянии ведите мышь по списку. Выделив необходимое количество записей, опустите клавишу мыши (клавишу Ctrl держите нажатой), подведите ее указатель к новому массиву записей и повторите выделение тем же способом и т. д.

Чтобы снять выделение массива щелкните по списку в любом месте. При этом выделенной будет одна строка, по которой был произведен последний щелчок.

Узнать количество выделенных записей можно с помощью свойства SelCount. Определить выделена ли запись с номером i можно по значению свойства Selected[i]: если оно равно True, то эта запись выделена.

Компонент ComboBox

Комбинированный список (ComboBox) – весьма удобное средство для создания списков с дополнительными (но необязательными) возможностями редактирования. Компонент объединяет строку ввода (аналогично компоненту Edit) с компонентом списка строк ListBox. При этом пользователи в строке ввода могут вводить новые строки.

Существует три основных стиля ComboBox:
Простой (Style = csSimple). Список всегда видимый. Пользователь может выбрать существующий элемент или ввести новый в строке ввода.
Раскрывающийся (Style = scDropDown). Список раскрывается, когда пользователь щелкает по кнопке, расположенной рядом со строкой ввода. Как и в случае стиля csSimple, пользователь имеет возможность вызова элемента в строку ввода или вводить новый элемент в этой строке.
Неизменяемый (Style = scDropDownList). То же, что scDropDown с той лишь разницей, что пользователь может щелчком вызвать элемент в строку ввода, но не имеет возможности ее редактирования.

Кроме приведенных стилей ComboBox имеет два производных от scDropDownList стиля – csOwnerDrawFixed и csOwnerDrawVariable, которые позволяют определтиь стиль списка по желанию разработчика.

ComboBox не имеет полосы прокрутки.

При разработке приложения вводите элементы в свойство Items объекта ComboBox.

Введите или очистите значение свойства Text, которому Delphi автоматически присваевает имя самого компонента ComboBox как значение.

Если Вы желаете получить в строковую переменную S строку из строки ввода объекта с именем ComboBox1, используйте, например, такой оператор:

S:= ComboBox1.Text;

Если необходимо скопировать в строку S пятый элемент списка CmBx (нумерация от 0), то можно использовать оператор:

S:= CmBx.Items[5];

Чтобы вставить в список строку из его строки ввода создайте событие OnKeyPress компонента ComboBox1 и вставьте в него текст:

if Key = #13 then begin
ComboBox1.Items.Add(ComboBox1.Text);
Key:= #0;
end;

Теперь при вводе строки каждое нажатие клавиши Enter (эта клавиша имеет код #13) приведет к вставке строки в список.

Для того чтобы открыть или закрыть список под управлением программы, присвойте свойству DroppedDown значение True или False, соответственно. Разместите, например, на форме комбинированный список ComboBox1 и какую-нибудь кнопку. Используйте следующий оператор в обработчике события OnClick для этой кнопки:

With ComboBox1 do
DroppedDown:= not DroppedDown;

Теперь при щелчке по кнопке закрытый список будет открываться, а открытый – закрываться.

Комбинированные списки являются объектами с единичным выбором. Лишь компоненты ListBox позволяют делать множественный выбор.

Вызов метода Clear приведет к удалению всех элементов списка и любого текста в строке ввода. Например, чтобы очистить объект ComboBox1 используйте следующий оператор:

ComboBox1.Clear;

Если Вы хотите программно очистить строку ввода, присвойте свойству Text пустую строку:

ComboBox1.Text:= '';

Для того чтобы добавить в список новую строку, используйте метод Add свойства Items

ComboBox1.Items.Add('Сидоров И.П.');

При выполнении этого оператора строка 'Сидоров И.П.' будет добавлена в конец списка ComboBox1.

Классы списков

Объекты ListBox и ComboBox являются визуальными компонентами представления списков в окнах. Однако большинство программ часто нуждаются в использовании списков, которые требовали бы меньшую память и высокую скорость обработки. К числу таких структур относятся три класса списков:
TList. Класс списков общего назначения.
TStrings. Абстрактный класс, используемый многими свойствами компонентов для обработки списков строк и других объектов. Эти объекты не занимают места в памяти при хранении.
TStringList. Класс, который используется для создания списков строк и объектов. Списки занимают память при хранении.

Приведенные структуры обычно играют вспомогательную, однако исключительно важную роль при разработке профессиональных приложений.

Наиболее часто в практике программирования используется один из них – класс TStringList. Обычно его используют при работе с текстовыми файлами для чтения или записи строк.

Приведем пример использования этого класса для решения следующей задачи.

Задача 1. Текстовый файл 'detals.txt' содержит строки, в которых представлены наименования деталей. Необходимо создать новый файл 'bearings.txt', который будет содержать в отсортированном виде те строки файла 'detals.txt', в которых присутствует слово подшипник.

Для решения задачи можно использовать следующую процедуру:

procedure CreateBearFile(InFile,OutFile,KeyWord: String);
var
i: Word; // счетчик цикла
SL1,SL2: TStringList; // описание списков
begin
SL1:= TStringList.Create; // создание входного списка
SL2:= TStringList.Create; // создание выходного списка
try
SL1.LoadFromFile(InFile); // чтение всех строк в SL1
if SL1.Count>0 then
begin // в списке SL1 строки есть
// Проход по строкам SL1
for i:=0 to SL1.Count-1 do
if Pos(AnsiUpperCase(KeyWord),
AnsiUpperCase(SL1.Strings[i]))>0 then
SL2.Add(SL1.Strings[i]); // добавление строки в SL2
// конец прохода
if SL2.Count>0 then // в выходном списке строки есть
begin
SL2.Sort; // сортировка SL2
SL2.SaveToFile(OutFile); // запись SL2 в выходной файл
end;
end; { if SL1.Count>0 }
finally
SL2.Free; // освобождение памяти, занятой SL2
SL1.Free; // освобождение памяти, занятой SL1
end; // finally
end; // процедуры

Процедура CreateBearFile решает поставленную задачу в общем виде, т. е. может обрабатывать и создавать файлы с любыми именами и искать строки с любой последовательностью символов. В качестве ее параметров использованы: InFile – исходный файл, OutFile – выходной файл, KeyWord – икомая последовательность символов в строках файла.

Для решения поставленной задачи достаточно обратиться к этой поцедуре оператором:

CreateBearFile('detals.txt','bearings.txt','подшипник');

При таком обращении процедура будет искать строки в файле InFile = 'detals.txt', записывать их в файл OutFile = 'bearings.txt', в качестве ключевого слова используется подстрока KeyWord = 'подшипник'.

В начале процедура CreateBearFile создает два списка SL1 и SL2 при помощи метода Create. Первый список предназначен для чтения из файла всех его строк, второй – для фильтрации в него строк из SL1, отвечающих критерию отбора. После создания списки размещаются в оперативной памяти. Для чтения файла использован метод LoadFromFile.

После чтения выполняется проверка количества прочитанных строк. Если в списке присутствуют строки (SL1.Count>0), то далее с помощью оператора цикла for в каждой строке последовательно проверяется наличие подстроки KeyWord с помощью функций Pos и AnsiUpperCase.

Функция Pos(s1, s2) дает положительное значение в том случае, если подстрока s1 полностью содержится в строке s2. Например,

Pos('Иван','Иванов')>0, Pos('Петр','Иванов')=0.

Функция AnsiUpperCase "поднимает" в верхний регистр все смволы ее аргумента. Например,

AnsiUpperCase('Ива'),AnsiUpperCase('ива'),AnsiUpperCase('ИвА')

возвратит одно и то же значение 'ИВА'.

Использование этих функций позволит найти все строки, в которых есть подстрока 'подшипник', независимо о того, как она записана в разных строках файла. Если бы функция AnsiUpperCase не была использована, то, например, строку, содержащую последовательность символов 'Подшипник', процедура не обнаружила бы, т. к. эта строка не совпадает с ключевой строкой 'подшипник' (отличие по первой букве).

Далее, если текущая строка из списка SL1 отвечает критерию отбора, то она добавляется в список SL2 методом Add.

По окончании отбора производится проверка количества строк в списке SL2. Если в нем есть отфильтрованные из SL1 строки, то список сортируется (строки располагаются в алфавитном порядке) с помощью метода Sort и затем записывается в выходной файл методом SaveToFile.

После выполнения этих операций в блоке finally – end списки, как объекты размещенные в памяти, освобождаются методом Free и процедура заканчивает работу.

П р и м е ч а н и е. При создании собственных объектов в памяти всегда используйте блок try – finally – end. Он гарантирует, что, если по какой-либо причине при работе программы возникнет исключительная ситуация, то объекты, под которые Вы выделяли память, будут освобождены.

Задача 2. Решить задачу 1 для отбора по следующим ключевым словам: болт, гайка, шайба. Результат фильтрации поместить в файл 'filters.txt '

Для решения этой задачи в целях ускорения работы компьютера следовало бы написать другую, независимую процедуру. Однако задачу можно решить, незначительно проиграв в скорости, используя уже созданную нами процедуру CreateBearFile. Для этого достаточно создать итоговый и рабочий списки, последовательно создать и загрузить в рабочий список три файла, каждый из которых отвечает своему ключевому слову и создается с помощью процедуры CreateBearFile, добавить каждый список в итоговый, отсортировать последний и записать в выходной файл.

Описанные действия сведены ниже в процедуру:

procedure Create3(InFile,OutFile: String);
var
SI,SF: TStringList; // описание списков
begin
SI:= TStringList.Create; // создание итогового списока
SF:= TStringList.Create; // создание вспом.списка
try
CreateBearFile(InFile,OutFile,'болт'); // фильтр 1
SF.LoadFromFile(OutFile);
SI.AddStrings(SF); // теперь содержит данные по 'болт'
CreateBearFile(InFile,OutFile,'гайка'); // фильтр 2
SF.LoadFromFile(OutFile);
SI.AddStrings(SF); // теперь содержит данные по 'болт' и 'гайка'
CreateBearFile(InFile,OutFile,'шайба'); // фильтр 3
SF.LoadFromFile(OutFile);
SI.AddStrings(SF); // теперь содержит данные по 'болт','гайка'и 'шайба'
SI.Sort; // сортировка итогового списка
SI.SaveToFile(OutFile); // запись итогового списка в файл
finally
SI.Free; // освобождение памяти
SF.Free;
end;
end;

Эта процедура решает задачу 2, однако выглядит "неуклюже", поскольку содержит три практически повторяющихся блока. Правильнее будет представить их одной универсальной процедурой, к которой можно обратиться трижды с учетом критерия фильтрации записей.

Ниже даны процедуры MakeSF и Create3M, которые решают задачу с учетом этого замечания.

Первая процедура, используя вспомогательный список SF, создает список по одному ключу, затем добавляет его в итоговый список SI. Вторая процедура, трижды обратившись к первой, создает, сортирует этот список и затем добавляет его в итоговый файл:

procedure MakeSF(InFile,OutFile:String; Key: String; var SI:TStringList);
var SF: TStringList;
begin
SF:= TStringList.Create;
Try
CreateBearFile(InFile,OutFile,Key);
SF.LoadFromFile(OutFile);
SI.AddStrings(SF);
finally SF.Free; end;
end;

procedure Create3M(InFile,OutFile: String);
var SI: TStringList;
begin
SI:= TStringList.Create;
Try
MakeSF(InFile,OutFile,'болт',SI);
MakeSF(InFile,OutFile,'гайка',SI);
MakeSF(InFile,OutFile,'шайба',SI);
SI.Sort;
SI.SaveToFile(OutFile);
finally SI.Free; end;
end;

Теперь, чтобы решить задачу 2 с помощью модернизированной процедуры Create3M, достаточно выполнить оператор:

Create3M('detals.txt','filters.txt');
