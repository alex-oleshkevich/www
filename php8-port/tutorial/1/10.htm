СОБЫТИЯ ВИДА "ПЕРЕТАЩИТЬ И ОПУСТИТЬ"

В среде Delphi имеется инструментарий, позволяющий программировать события, объединенные общим термином drag and drop (перетащить и опустить). Эти события происходят при буксировке одного объекта по другому с помощью мыши. Многие визуальные компоненты "распознают" событие OnDragOver (на волочение), которое имеет место тогда, когда пользователь буксирует поверх объекта другой объект. Обычно это событие устанаваливает свойство Accept компонента-приемника в True или False, в зависимости от того может или нет этот объект принять буксируемый объект. Имеется еще одно событие, играющее не менее важную роль, OnDragDrop (на опускание), которое происходит в тот момент, когда пользователь опускает на объект-приемник буксируемый объект. При необходимости можно использовать событие OnEndDrag (на окончание волочения), которое отностся к буксируемому объекту. Оно возникает вне зависимости от того, был компонент опущен или нет. Здесь важен сам факт начала буксировки.

Приложение DragDrop

Проиллюстрируем обработку событий drag and drop на примере приложения, которое позволит из карточек с надписанными на них буквами составлять слова, путем перетаскивания нужных букв из исходного набора в зону формирования слова. Наряду с этим запрограммируем приложению простейший интеллект: оно позволит из вынутых карточек сложить слово, а в некоторых случаях даже толковать его смысл.

Конструирование формы

Запустите Delphi. Если в ней находится приложение, закройте его.

Создайте новое приложение. Измените значение свойства Name формы на fMain, свойства Caption – на Составитель и толкователь слов, свойства Position – на poDesktopCenter. Сохраните модуль в новой папке DragDrop под именем main.pas, проект – под именем DragDrop.dpr. Эта папка должна находиться внутри Вашей общей папки, в которой содержатся папки ранее созданных Вами приложений.

Теперь необходимо нанести на форму необходимые визуальные компоненты. Вид готовой формы показан на рис. 9.1.

Сначала положите на форму компонент Panel ( закладка Standard). На рисунке она расположена по середине формы и занимает почти всю ее площадь. Измените значение свойства Caption метки на прописную букву А. Щелкните по значению свойства Font метки, затем – по маленькой кнопке с троеточием. В открывшемся окне Выбор шрифта измените размер на 24, цвет – белый. Закройте окно редактора шрифта кнопкой ОК. Измените значение Color (цвет подложки) метки на clNavy (цвет морской волны). Теперь компонент Label1 должен выглядеть как карточка с надписанной буквой А (см. рис. 9.1).

Программирование событий

Сначала дадим меткам-карточкам возможность буксировки по панели. Для этого нажмите клавишу Shift и последовательно щелкните по всем этим меткам (они должны выделиться серыми квадратиками). В окне Object Inspector измените значение свойства DragMode на dmAutomatic. Теперь все карточки могут буксироваться с помощью мыши.

Далее запрограммируем событие буксировки. Щелкните по панели Panel1, в окне Object Inspector щелкните по закладке Events, затем – по значению события OnDragOver. В заготовку обработчика события внесите следующий оператор:

Accept:= Source is TLabel;

Этот оператор "разрешит" при буксировке по панели опустить на нее любую метку.

Далее создадим событие, возникающее при опускании карточки на панель. В окне Object Inspector щелкните по значению события OnDragDrop и внесите в обработчик текст:

With Source as TLabel do begin Left:=x; Top:=y; end;

При исполнении этих операторов опускаемая на панель метка получит координаты верхнего левого угла, равные координатам указателя мыши в тот момент, когда пользователь отпустит ее левую клавишу. По сути эти операторы "заставляют" мышь волочить карточку за ее левый верхний угол.

Запустите программу и убедитесь, что все карточки можно перемещать по панели. Обратите внимание, что в момент нажатия клавиши мыши в зоне карточки курсор мыши превращается в перечеркнуый круг. Это означает, что опускать метку на себя нельзя. Когда курсор сдвигается с карточки на панель, то приобретает форму стрелки с листом. Это значит, что буксируемый объект можно опустить на панель. При переходе через фаску курсор "не разрешает" опускание, а при входе в зону выкладывания карточек вновь "разрешает" положить буксируемую карточку на панель. Выложите из карточек какое-нибудь слово. Убедитесь, что кнопки Button еще не действуют. Для выхода из программы и возврата в среду Delphi щелкните по кнопке с надписью Выход.

Далее создадим отдельную процедуру Clear:

procedure Clear;
begin
With fMain do
begin
Word:='';
LWord.Caption:='';
LSense.Caption:='';
bSense.Enabled:=false;
end;
end;

Процедура использует глобальную строковую переменную Word, которая предназначена для накопления букв, составляющих выложенное слово. Оператор описания этой переменной расположен сразу вслед за оператором описания формы fMain в секции var (листинг 9.1).

Из текста видно, что процедура очищает переменную Word, надписи на кнопках LWord и LSense (присваевает им пустую строку символов), а кнопку bSense делает недоступной. Эта процедура будет использована трижды: один раз при активизации формы, второй – в начале процесса распознавания выложенного слова и третий – при приведении формы в исходное состояние, соответствующее началу манипуляций по выкладыванию нового слова. Объясняется это тем, что в этих ситуациях надписей с распознанным словом и его смыслом не должно быть на экране, а поскольку нет слова для толкования его смысла, то кнопка предназанченная для выпонения процедуры толкования, не должна быть доступной пользователю.

Для выполнения ряда подготовительных операций создайте для формы обработчик события OnActivate (на активизацию):

procedure TfMain.FormActivate(Sender: TObject);
var L: Integer;
begin
Clear;
SetLength(Arr,ComponentCount);
for L:=0 to ComponentCount-1 do
if Components[L] is TLabel then
With Components[L] as TLabel do
begin
Arr[L].x:=Left;
Arr[L].y:=Top;
end;
end;

В теле этого обработчика сначала происходит обращение к вышеописанной процедуре Clear, затем выполняется ряд операторов с целью запоминания исходного положения карточек в динамическом одномерном массиве Arr, который, как и Word, представляет собой глобальную переменную. Описание этого массива дано в секции var (листинг 9.1). Оператор SetLength устанавливает в памяти размер массива равным количеству визуальных компонентов на форме (ComponentCount). Затем выполняется цикл с целью прохода по всем компонентам. Если текущий компонент является меткой (Components[L] is TLabel), то элемент массива Arr, соответствующий номеру компонента, получает значения координат левого верхнего угла этой метки. Таким образом, в массиве окажутся зафиксированными координаты не только карточек, но и всех других меток. Конечно, это излишняя трата памяти и времени. Однако если учесть минимум и простоту программного кода и ничтожное количество перерасходованной памяти (несколько байтов), то такое решение следует признать наилучшим !

Поскольку массив Arr является динамическим, т. е. память под него выделяется в процессе работы программы, то по завершении ее работы эту память необходимо освободить.

Создайте для формы обработчик события OnClose (на закрытие формы) и внесите в него оператор

Arr:= Nil;

освобождения памяти, выделенной под массив процедурой SetLength в обработчике OnActivate.

Запустите программу и убедитесь, что надписи на метках LWord и LSense исчезли, а кнопка с надписью Что означает … стала недоступной. Это внешнее проявление сработавшей процедуры Clear, вызванной при выполнении обработчика OnActivate. Кроме того сработали операторы запоминания исходного положения всех меток (внешне это никак не проявилось). Закройте приложение (память под Arr освободилась при закрытии формы, внешне это также не проявилось).

Теперь запрограммируем действие Button-кнопок.

Первая из них (она с именем bWord) должна запускать процедуру, которая решает две задачи: складывает слово из вынутых карточек и передает его в метку LWord с тем, чтобы оно стало видно на форме.

Щелкните дважды по этой кнопке и создайте для нее обработчик события OnClick (на щелчок):

procedure TfMain.bWordClick(Sender: TObject);
var i,L: Integer;
begin
Clear;
for i:=1 to Panel1.Width do
for L:=0 to ComponentCount-1 do
if Components[L] is TLabel then
With Components[L] as TLabel do
if (Top>Bevel1.Top) and (Top+Height<Bevel2.Top) then
if i = Left then Word := Word + Trim(Caption);
LWord.Caption:=Word;
bSense.Enabled:= Word <> '';
end;

При запуске этого обработчика сначала срабатывает процедура Clear.

Далее производятся следующие действия. В цикле для каждого значения переменной i, меняющей его от 1 до значения, равного ширине панели (в пикселах), выполняется еще один цикл (по переменной L), который обеспечивает проход по всем компонентам формы. Если компонент является меткой, то проверяется находится ли его вершина ниже вершины верхней линии (Top>Bevel1.Top), а дно – выше вершины нижней линии (Top+Height < Bevel2.Top), другими словами, находится ли карточка в зоне выкладываемого слова. Если это так (т. е. это – карточка и она – в зоне слова), то далее проверяется условия совпадения номера текущего горизонтального пиксела панели с номером пиксела, в котором расположен верхний угол этой карточки. Проще говоря, происходит следующее. Берется первый столбец пикселов панели и последовательно проверяются все визуальные компоненты на принадлежность их к типу "метка", потом – на принадлежность зоне выкладывания слова, потом – на принадлежность левого угла метки этому столбцу. Невыполнение любого из условий означает, что подходящая карточка еще не найдена. Далее производится смещение области поиска на второй столбец пикселов панели и так до тех пор, пока не будет найдена карточка, удовлетворяющая всем этим условиям. Понятно, что если будут пройдены все столбцы пикселов панели и не будет ныйдено ни одной такой карточки, то слова в зоне его выкладывания нет. Обратите внимание, что алгоритм реагирует только на те карточки, которые полностью лежат в зоне выкладывания слова.

Если карточки в зоне есть, то сначала будет найдена та из них, которая расположена левее других. Теперь исполнится оператор

Word := Word + Trim(Caption);

который к пустой строке Word добавит надпись на этой карточке (Caption) и результат запишет в эту же переменную, т. е. если на карточке надписана, например, буква А, то значением переменной Word теперь будет строка 'А'. Обратите внимание, что для страховки от проникновения пробелов в формируемое слово использована функция Trim, которая отбрасывет лидирующие и завершающие пробелы аргумента. Аналогично находится следующая карточка, которая правее первой, но левее остальных. Далее к старому значению Word добавляется новая буква. Например, если на второй карточке надписана буква Ж, то значением Word теперь будет строка 'АЖ'. Процесс будет продолжаться до тех пор, пока не будет сканирована вся поверхность зоны слова. Понятно, что по его окончании значением переменной Word будет строка, составленная из букв карточек, находящихся в зоне слова.

По окончании этого процесса оператором

LWord.Caption:= Word;

полученное слово будет передано в надпись метки LWord. В результате, если процедура обнаружила карточки, то рядом с надписью Слово: на форме появится это слово. Если карточек нет, то строка Word и надпись на форме будут пустыми (вспомните Clear).

Последним действием обработчика будет решение вопроса о доступности кнопки bSense:

bSense.Enabled:= Word <> '';

означающее, что если Word непусто, то кнопку следует сделать доступной для пользователя с тем, чтобы для непустого слова была создана возможность к его толкованию. Если слова не обнаружено, то кнопка bSense останется по-прежнему недоступной.

Перейдем к программированию события OnClick от щелчка по кнопке bSense (с надписью Что означает …). Дважды щелкните по кнопке и создайте обработчик события – процедуру:

procedure TfMain.bSenseClick(Sender: TObject);
var i: integer;
begin
if Word <> '' then
begin
LSense.Caption:='Не знаю';
for i:= 0 to LB1.Items.Count-1 do
if Trim(LB1.Items.Strings[i]) = Word then
begin
bSense.Enabled:=true;
LSense.Caption:=LB2.Items.Strings[i];
break;
end;
end;
end;

Эта процедура предназначена для определения смысла выложенного слова. Идея алгоритма заключается в следующем. Если слово непусто, то сначала ответ формируется в виде строки 'Не знаю'. Далее совершается проход по словам списка LB1 и если какая-нибудь его строка совпадет со строкой Word, содержащей выложенное слово, то надпись на кнопке LSense будет заменена строкой с тем же индексом из списка LB2, содержащего толкование слова из списка LB1. Если же в списке LB1 не найдется выложенного слова, то надпись на кнопке LSense останется прежней (т. е. 'Не знаю').

На рис. 9.2 приведена ситуация, когда пользователь выложил слово из карточек c буквами Е и Ж После щелчка по кнопке bWord (с надписью Составлено слово …) программа, проанализировав панель процедурой TfMain.bWordClick, отреагировала, выдав слово ЕЖ.

Рис.9.2. Программа DragDrop дает толкование выложенного слова

Далее после щелчка по кнопке bSense (с надписью Что означает …) была выполнена процедура TfMain.bSenseClick, результатом работы которой стала выведенная на форму надпись Животное с иголками.

На рис. 9.3. приведен рабочий вариант программы, когда на выложенное слово ДОМ, дан ответ Не знаю. Объяснение очевидно – список LB1 не содержит этого слова, а следовательно, в списке LB2 нет строки-смысла. В таких случаях процедура всегда дает одинаковый отрицательный ответ.

Обратите внимание, что карточки с буквами Г, А программой в расчет не приняты, т. к. расположены выше или ниже зоны выкладывания слова. Обратите также внимание на пункирный контур на кнопке bSense: это значит, что последней была нажата именно эта кнопка.

Рис.9.3. Программе DragDrop не известен смысл выложенного слова

Программирование кнопок завершим созданием события от щелчка по кнопке bReset (с надписью Сброс):

procedure TfMain.bResetClick(Sender: TObject);
var L: Integer;
begin
Clear;
for L:=0 to ComponentCount-1 do
if Components[L] is TLabel then
With Components[L] as TLabel do
begin
Left:=Arr[L].x;
Top:=Arr[L].y;
end;
end;

Процедура предназначена для приведения формы в исходное состояние. Из текста процедуры видно, что сначала обращением к процедуре Clear производится очистка надписей и кнопка толкования делается недоступной, а затем в цикле по переменной L всем меткам возвращаются координаты, сохраненные при активизации формы в массиве Arr. При щелчке по кнопке bReset это внешне проявится, в частности, в том, что все сдвинутые с исходного положения карточки займут исходные места.

procedure TfMain.FormActivate(Sender: TObject);
var L: Integer;
begin
Clear;
SetLength(Arr,ComponentCount);
for L:=0 to ComponentCount-1 do
if Components[L] is TLabel then
With Components[L] as TLabel do
begin
Arr[L].x:=Left;
Arr[L].y:=Top;
end;

X,Y: Integer; State: TDragState; var Accept: Boolean);
begin
Accept:= Source is TLabel;
end;

begin
With Source as TLabel do begin Left:=x; Top:=y; end;
end;

procedure TfMain.bWordClick(Sender: TObject);
var i,L: Integer;
begin
Clear;
for i:=1 to Panel1.Width do
for L:=0 to ComponentCount-1 doif Components[L] is TLabel then
With Components[L] as TLabel do
if (Top>Bevel1.Top) and (Top+Height<Bevel2.Top) then
if i = Left then Word := Word + Trim(Caption);
LWord.Caption:=Word;
if Word <> '' then bSense.Enabled:= true;
end;

var L: Integer;
begin
Clear;
for L:=0 to ComponentCount-1 do
if Components[L] is TLabel then
With Components[L] as TLabel do
begin
Left:=Arr[L].x;
Top:=Arr[L].y;
end;
end;

Arr:= Nil;

begin
if Word <> '' then
begin
LSense.Caption:='Не знаю';

begin
bSense.Enabled:=true;
