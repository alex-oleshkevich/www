5. ДИАЛОГОВЫЕ ОКНА ВВОДА-ВЫВОДА ТЕКСТА

Весьма часто при при создании приложений возникает необходимость ввода или вывода текста в окне, которое появляется над рабочей формой и, выполнив свою роль, по команде пользователя исчезает с экрана. Примерами окон вывода могут быть сообщения с реакцией программы о неверно введенных пользователем данных, разного рода предупреждения, напоминания, окна с вопросами, от ответа на которые со стороны пользователя зависит направление, в котором будет развиваться процесс. Примерами окон ввода являются окна ввода пароля или какой-нибудь строки текста, которая программе необходима в данной ситуации.

Окна вывода

В Delphi имеется несколько окон вывода – от простых до многовариантных. Простое окно обычно выводит какое-нибудь сообщение в виде предупреждения или напоминания. Многовариантное окно позволяет запрограммировать ситации для разветвляющихся алгоритмов.

Простое окно ShowMessage

Это окно созается обращением к процедуре ShowMessage. Описание процедуры имееет вид :

procedure ShowMessage(const Msg: string);

где Msg – строка, которую необходимо поместить в окно. Окно позиционируется по середине экрана монитора.

Например, оператор

ShowMessage('Ошибка в данных …');

выведет окно, показанное на рис. 5.1.

Многовариантное окно MessageDlg

Значительно большие возможности по сравнению с ShowMessage имеет окно MessageDlg.

Общий вид заголовка функции, формирующей окно:

function MessageDlg(const Msg: string; AType: TMsgDlgType;
AButtons: TMsgDlgButtons; HelpCtx: Longint): Word;

Здесь
Msg – выводимое сообщение.
AType – тип собщения. В роли типа могут выступат константы: mtWarning (предупреждение), mtError (ошибка), mtInformation (информация), mtConfirmation (подтверждение). Типу сообщения на окне будет соответсвовать картинка с изображением: для mtWarning – желтый предупредительный треугольник с восклицательным знаком, для mtError – красный круг с крестиком, для mtInformation – пузырек с буквой i, для mtConfirmation – пузырек со значком вопроса.
AButtons – список кнопок, которые необходимо поместить на окно. Например, для размещения кнопок Yes (Да), No (Нет), Cancel (Отменить), Ignore (Игнорировать), Retry (Повторить) нужно передать в заголовок строку [mbYes, mbNo, mbCancel, mbIgnore, mbRetry].
HlpCtx – индикатор контекстной помощи. Если для окна не определен раздел в контекстно-зависимой справочной системе, то параметр следует положить равным нуля.

Функция возвращает целое, определяющее кнопку, на которую нажал пользователь для закрытия окна. Окно позиционируется по середине экрана.

На рис. 5.2 показано окно, которое сформировано с помощью оператора

MessageDlg('Ошибка в данных', mtInformation, [mbOk], 0);

В этом примере обращение к функции выполнено в форме обращения к процедуре: окно предполагает одновариантный ответ, поэтому для упрощения допустимо такое обращение.

В более сложных случаях обращение должно иметь другой вид.

Очень часто окно MessageDlg используется программистами для создания собственной функции для получения от пользователя ответа в форме Да/Нет. Если положить на окно две кнопки Yes и No и запрограммировать получение ответа от нажатия на одну из кнопок, то этого может оказаться недостаточно: пользователь может щелкнуть по крестику в правом углу окна, что следует трактовать как отрицательный ответ на поставленный вопрос, т. е. как разновиднось щелчка по кнопке No. Вот пример функции, которая может быть использована для получения ответа от пользователя:

Function DaNet(S: String): boolean;
begin
Result:= MessageDlg(S,mtConfirmation,[mbYes,mbNo],0) = mrYes;
Screen.ActiveForm.Refresh;
end;

Первый оператор в теле функции показывает, что окно содержит две кнопки. Ответ считается положительным, если пользователь щелкнул по кнопке Yes (функция возвратит результат, равный константе mrYes). В остальных вариантах (щелчок по кнопке No или по крестику) ответ будет признан отрицательным. В этой функции после получения ответа на вопрос производится выполнение дополнительного оператора обновления (Refresh) формы, которая является активной в момент выпадания окна с вопросом. Оператор использован для того, чтобы убрать с экрана следы исчезнувшего окна, которые иногда остаются после его закрытия.

Следующий пример демонстрирует ситуацию, когда программа готова удалить папку с файлами:

if DaNet('Удалить папку ?') then
if DaNet('Вы действительно хотите удалить папку ?') then
DeleteFolder;

Рис.5.3. Вид окна вывода вопроса при обращении к функции DaNet

Операция удаления данных относится к числу тех опасных действий, которые могут привести к непоправимым ошибкам. По этой причине фрагмент программы составлен так, что вопросы по поводу удаления папки задаются дважды. Первый раз – в виде простого вопроса. Получив положительный ответ, программа с учетом усиления опасности, в более настойчивой форме просит пользователя еще раз дать ответ на поставленный вопрос. Только получив вторичное подтверждение, программа уничтожит папку.

Можно было бы ограничиться одним вопросом, однако психология среднестатистического пользователя такова, что вне зависимости от сложности ситуации он может машинально дать нежелательный ответ. Здесь положение можно поправить, лишний раз заострив его внимание на поставленном вопросе. Конечно, такой подход к обработке сложных ситуаций украшает программу, однако в менее ответсвенных ситуациях многократное повторение вопросов по одному и тому же поводу может вызвать у пользователя раздражение, что следует расценивать уже как недостаток программы.

Обратите внимание на простоту обращения к функции DaNet: она содержит минимум атрибутов – имя функции и строку с вопросом. Прочие атрибуты, учитывающие особенности окна вывода "спрятаны" внутри тела функции DaNet. Действительно, если бы эти же вопросы пришлось задавать с помощью исходной функции MessageDlg, то код прибрел бы более сложный вид (для выполнения кода использована дополнительная логическая переменная b):

b:= MessageDlg('Удалить папку ?',
mtConfirmation,[mbYes,mbNo],0) = mrYes;
Screen.ActiveForm.Refresh;
if b then
begin
b:=MessageDlg('Вы действительно хотите удалить папку ?',
mtConfirmation,[mbYes,mbNo],0) = mrYes;
Screen.ActiveForm.Refresh;
if b then DeleteFolder;
end;

П р и м е ч а н и е. Практикующие программисты обычно создают отдельные модули по категориям назначения, в которые помещают собственные процедуры и функции, часто использующиеся в стандартных ситуациях. Функция DaNet, являет собой типичный пример таких алгоритмов.

Многовариантное окно MessageDlgPos

Это окно является практически полной копией окна MessageDlg. Разница заключается только в том, что его можно позиционировать в любом месте экрана.

Общий вид функции, воспроизводящей окно:

function MessageDlgPos(const Msg:string; AType:TMsgDlgType;
AButtons:TMsgDlgButtons; HelpCtx:Longint; X,Y:Integer):Word;

где x,y – координаты верхнего левого угла окна, прочие параметры полностью совпадают с соответствующими параметрами функции MessageDlg.

Приведем пример использования окна, которое будет выведено в левом верхнем углу экрана монитора:

if MessageDlgPos('Закончить работу с массивом ?',
mtConfirmation,[mbYes,mbNo],0,1,1) = mrYes
then r:= Nil;

Обычно программисты отдают предпочтение центрированным окнам, поэтому мы не станем останавливаться на особенностях использования окна MessageDlgPos.

Окно ввода текста

Диалоговые окна вывода предоставляют пользователю информацию. Для получения данных от пользователя используют окна ввода. В Delphi есть две функции, которые отображают такие окна и обеспечивают ввод строк – функции InputBox и InputQuery.

Окно InputBox

Описание функция InputBox имеет следующий вид:

function InputBox(const Header,Prompt,Default: string): string;

Функция имеет строковый тип, а ее аргументами являются строки:
Header – надпись в заголовке окна,
Prompt – поясняющая надпись,
Default – исходное состояние строки ввода.

Например при выполнении операторов

s:= 'Сергеева';
s:= InputBox('Окно ввода','Исправьте фамилию студента',s);

на экране появится диалоговое окно ввода, представленный на рис. 5.4.

В заголовке окна отображена надпись Окно ввода. Ниже заголовка, но выше строки ввода находится поясняющая надпись Исправьте фамилию студента, в строке ввода – текст Сергеева, ниже ее расположены две кнопки. Клавиатурный курсор позиционирован в конце строки ввода.

Рис.5.4. Вид окна InputBox ввода строки

Теперь пользователь может исправить фамилию или оставить ее без коррекции и нажать одну из кнопок. Если будет нажата кнопка OK, то переменная s получит значение, равное той строке, которая находится в окне ввода. Если пользователь щелкнет по кнопке отмены, крестику или нажмет клавишу Esc, то значение переменной s останется таким, каким оно было до обращения к функции, вне зависимости от того, была изменена строка в окне или нет ( в нашем примере строка Сергеева).

Окно InputQuery

Функция InputBox может удовлеворить многие запросы, однако она не содержит сведений о том были ли произведены изменения при вводе данных. Этого недостатка лишена другая, похожая на нее функция InputQuery:

function InputQuery(const Header,Prompt: string;
var Value: string): Boolean;

В отличие от InputBox эта функция имеет логический тип, а значение строки ввода находится в переменной Value. Если пользователь нажал кнопку ОК и строка действительно была изменена, то функция возвратит True, иначе – False. Внешне это окно ничем не отличается от окна InputBox.

Приведем пример.

ReadFromFile(nF,i,s,31);
if InputQuery('Ввод строки','Введите новые данные',s)
then WriteToFile(nF,i,s,31);

Эти операторы читают в переменную s строку из файла nF с позиции i при помощи некой процедуры ReadFromFile. Далее пользователю предоставлена возможность исправить эту строку с помощью окна ввода InputQuery. Если после закрытия окна ввода в строке будут обнаружены коррективы, т. е. функция InputQuery возвратит значение True, то строка будет перезаписана в файле при помощи процедуры WriteToFile, иначе перезаписи не будет.
