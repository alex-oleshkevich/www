3. ПРОГРАММИРОВАНИЕ ДИАЛОГОВ

Для облегчения работы при выборе и сохранении файлов, для поиска текста, при выборе фонтов, цветов, для настройки принтеров и печати используют стандартные компоненты диалогов. Эти компоненты находятся на закладке Dialogs палитры визуальных компонентов.

Компоненты диалогов

Для того, чтобы понять действие диалогов рассмоторим новое приложение и разберем шесть диалоговых компонентов OpenDialog, SaveDialog, ColorDialog, FontDialog, FindDialog, ReplaceDialog. В качестве текстового редактора будет использован новый компонент RichEdit, который обладает значительно большими возможностями по сравнению с рассмотренным ранее компонентом Memo. В частности, этот компонент позволяет создавать текст, отдельные фрагменты которого могут иметь собственный фонт (шрифт) и цвет подложки. По этой причине при создании собственных редакторов текста программисты отдают предпочтение именно этому визуальному компоненту.

Приложение MyDialogs

Если Delphi еще не запущена, запустите ее. Если в ней содержится какое-либо приложение, закройте его командой File/Close All.

Теперь создайте новое приложение командой File/New Application. Cохраните этот модуль и приложение в целом в своей папке. Для этого нажмите кнопку SaveAll на панели инструментов. При сохранении модуля Unit1.pas создайте в выпавшем окне новую папку MyDialogs. Для этого двойным щелчком по папке войдите в нее, затем сохраните модуль Unit1.pas под именем Main.pas. Вслед за этим сохраните проект Project1.dpr под именем MyDialogs.dpr.

Форма, панели и компонент RichEdit

В окне Object Inspector измените свойство Position формы на poDesktopCenter. Это необходимо для того, чтобы при запуске приложения окно формы располагалось по центру экрана. Измените значение свойства Caption формы на Текстовый редактор приложения MyDialogs. Дайте форме имя fMain.

Уберите текст из свойства Caption верхней панели (щелкните по этой панели, затем в окне Object Inspector выполните удаление текста свойства Caption).

Расширенный редактор RichEdit

Положите на нижнюю панель компонент RichEdit, который находится на закладке Win32, и присвойте свойству Align значение alClient. При этом компонент накроет всю панель. Измените его имя (свойство Name) на Re. Уберите текст из редактора. Для этого войдите в свойство Lines, в выпавшем окне уберите весь текст и закройте окно кнопкой OK. Свойство ScrollBars сделайте равным ssBoth (показывать обе полосы прокрутки текста), а свойство PlainText – True.

Диалоговые компоненты

Перейдите на закладку Dialogs палитры компонентов и последовательно положите на компронент RichEdit компоненты OpenDialog, SaveDialog, ColorDialog, FontDialog, FindDialog, ReplaceDialog. Среда Delphi присвоит им имена OpenDialog1, SaveDialog1, ColorDialog1, FontDialog1, FindDialog1, ReplaceDialog1, соответственно. Вы можете изменить эти имена (свойство Name) на другие. Часто для того, чтобы уменьшить работу по составлению программы длинные имена меняют на короткие, но так, чтобы можно было бы по минимальным признакам идентифицировать компоненты и чтобы эти имена не пересекались с именами других переменных программы. Например, можно этим компонентам дать имена OD, SD, CD, FoD, FiD, RD, соответственно. В этом примере для большей наглядности имена компонентов оставлены такими, какие были даны им средой Delphi.

Прогаммирование диалоговых событий

Запустите приложение, щелкнув по кнопке Run панели инструментов Delphi (кнопка с зеленым треугольником) или нажав клавишу F9 на клавиатуре, и убедитесь, что в работающем приложении ни одна из кнопок SpeedButton не действует. Обратите внимание, что, если подвести курсор мыши к такой кнопке и задержать его, то под кнопкой появится подсказка (Hint) с текстом, отражающим назначение этой кнопки. Закройте приложение, щелкнув по кнопке Закрыть. Из опыта ранее рассмотренных приложений понятно, что кнопки не действуют из-за того, что еще не созданы обработчики событий, которые должны появляться при нажатии этих кнопок.

Создадим эти события.

У всех диалогов, визуальные компоненты которых в нашем приложении лежат на компоненте RichEdit, есть общий метод Execute (выполнить), который активизирует окно диалога, позволяя в нем изменить, выбирать или назначать необходимые параметры (для каждого диалога свои). Этот метод является логической функцией типа boolean.

Диалоговое окно открытия файла OpenDialog

Начнем создание обработчиков событий с кнопки sbOpenFile. Щелкнув по этой кнопке, выделите ее. Перейдите в окно Object Inspector и щелкните в нем по закладке Events (события). Теперь дважды щелкните по правому столбцу напротив надписи события OnClick (обработчик события – на однократное нажатие левой клавиши мыши). Delphi откликнется на это действие созданием заготовки обработчика события – процедуры, которую сразу можно увидеть в окне Code Editor модуля Main.pas. Эту заготовку можно сделать проще – двойным щелчком по кнопке. Вставьте между операторами begin и end заготовки следующий текст:

if OpenDialog1.Execute then
Re.Lines.LoadFromFile(OpenDialog1.FileName);

Чтобы понять действия, которые сопровождают выполнение этих операторов, запустите приложение. Далее нажмите кнопку с надписью "1".

В этом окне можно выбрать файл, текст которого Вы хотите видеть в редакторе RichEdit Вашего приложения. Выбор файла производится аналогично тому, как это делается при открытии файла в редакторе Microsoft Word: здесь можно найти требуемый файл на диске в текущей или другой папке и щелкнуть по его имени в списке (при этом имя файла должно появиться в окошке "Имя файла" диалогового окна). Если после выбора файла щелкнуть по кнопке Открыть, то это будет означать, что метод OpenDialog1.Execute следует выполнить. Если щелкнуть по кнопке Отмена, то метод Execute не следует выполнять.

Если необходимо, чтобы при открытии файла окно сразу показывало требуемую папку, то можно внести изменения в свойство InitialDir компонента OpenDialog1. На рис 3.3 показано как при открытии окна инициализировать корневой каталог диска C с помощью окна Object Inspector. Аналогично можно внести другие данные. Эти же данные можно инициализировать программно.

Таким образом, оператор Re.Lines.LoadFromFile(OpenDialog1.FileName) обработчика события будет выполнен, только тогда, когда будет истинным значение функции OpenDialog1.Execute. В этом случае строки (Lines) редак-тора Re будут загружены (LoadFromFile) тем файлом (FileName), который выбран при диалоге OpenDialog1. При нажатии кнопка Отмена в диалоговом окне состояние нашего редактора останется без изменения.


Кнопкой с надписью Закрыть на верхней панели формы закройте ее и приложение в целом для возврата в среду Delphi и продолжения работы по созданию событий.

Диалоговое окно сохранения файла SaveDialog

Теперь создадим обработчик события для кнопки sbSaveFile (с надписью "2") . Эта кнопка предназначена для сохранения файла, содержимое которого можно видеть в окне редактора Re. Сохранение необходимо тогда, когда содержимое редактора было изменено и измененный текст необходимо перезаписать на диск вместо ранее сохраненного текста. Если это не сделать, то дисковый файл останется таким, каким он был при последнем сохранении независимо от того вносились в текст редактора коррективы или нет. При необходимости содержимое редактора можно сохранить в другом файле. Таким образом, при сохранении файла в нашем примере будет происходить уничтожение старого содержимого файла и запись текущего содержимого редактора Re в этот файл.

Щелкнув по кнопке с надписью sbSaveFile, выделите ее. Перейдите в окно Object Inspector и по аналогии с созданием обработчика события для предыдущей кнопки создайте обработчик события для кнопки sbSaveFile.

Вставьте в заготовку процедуры-обработчика между операторами begin и end следующий оператор

if SaveDialog1.Execute then
Re.Lines.SaveToFile(SaveDialog1.FileName);

Вновь запустите приложение. Внесите в текстовый редактор следующий текст:

"Осталось решить, как процессор получает данные от внешних устройств и как он узнает, откуда данные пришли и где взять инструкции для их обработки.

Для этого используют так называемые прерывания. Существует 256 различных видов прерываний, которые могут прервать работу процессора над выполнением текущей программы и направить его на выполнение другой программы. Адрес этой программы процессор находит в специальной области памяти, которая называется вектором прерываний.

Допустим, мы решили подключить к компьютеру холодильник. Выберем для него какое-нибудь прерывание, не используемое другими устройствами, например "номер 179". Когда холодильник захочет обратить на себя внимание процессора, он пошлет на одну из его ножек специальный сигнал прерывания, а затем – число 179. Получив это число, процессор "заглядывает" в вектор прерываний и находит адрес программы, обслуживающей холодильник. Он перейдет по этому адресу и начнет работать с этой программой.'

После ввода этого текста щелкните по кнопке с надписью "2". При этом на экране появится диалоговое окно сохранения файла. Внешний вид его показан на рис. 3.4.

Диалоговое окно поиска текста FindDialog

Создайте далее обработчик события для кнопки sbFindText (с надписью "6") . Эта кнопка предназначена для поиска фрагмента текста в редакторе Re. Внесите в обработчик события текст

FindDialog1.Execute;

Этого однако недостаточно для поиска текста, т. к. обработчик лишь запускает процесс поиска, но программе пока неизвестно где искать текст. Чтобы дать понять приложению местоположение текста (в нашем случае это редактор Re), создайте еще один обработчик – на событие OnFind (на поиск) компонента FindDialog1 и введите в него текст так, чтобы процедура-обработчик имела вид:

procedure TForm1.FindDialog1Find(Sender: TObject);

var {объявление дополнительных переменных}
FoundAt: LongInt;
StartPos, ToEnd: integer;

Begin {начало процедуры-обработчика}
with Re do {искать текст будем в редакторе Re}
begin
if SelLength <> 0 then {в редакторе есть выделенный фрагмент}
StartPos:= SelStart + SelLength {нач.поз.выделенного текста}
else {если нет выделенного текста, то ищем от курсора}
StartPos:= 0;
ToEnd:= Length(Text)-StartPos; {искать до конца текста}
{теперь ищем текст}
FoundAt:= FindText(FindDialog1.FindText,StartPos,ToEnd,[stMatchCase]);
if FoundAt <> -1 then {если текст найден}
begin
SetFocus; {установка курсора в начало найденного текста}
SelStart:= FoundAt; {выделяем начало текста ...}
SelLength:=Length(FindDialog1.FindText); {... и его конец}
end
else
{если текст не найден, выдаем соответствующее сообщение}
ShowMessage('Текст не найден');
end;
end;

Щелкнув по кнопке Run панели Delphi, запустите приложение.

Найдите и откройте ранее сохраненный файл 1.rtf при помощи кнопки с надписью "1", затем – выбором в окне открытия файла.

После появления текста в редакторе нажмите кнопку с надписью "6" для поиска текста.

Щелкните по окошку напротив надписи "Образец" и введите текст, который хотите найти в редакторе. Например это может быть строка "179" (кавычки, разумеется, вводить не нужно).

Нажмите в окне поиска кнопку Найти далее. Программа запустит процесс поиска введенного текста. Если текст существует, то будет найдено и выделено место первого вхождения этого текста в редакторе.

Снова нажмите эту кнопку. Программа найдет следующее вхождение текста "179". В третий раз нажмите кнопку Найти далее. Теперь на экране появится окно с сообщением "Текст не найден", т. к. текст "179" встречается в редакторе только 2 раза. Чтобы возобновить поиск этого или другого текста, уберите окно поиска, щелкнув по кнопке Отмена. Щелчком в редакторе установите клавиатурный курсор в то место, начиная с которого необходимо выполнить новый поиск, и снова запустите процесс поиска кнопкой с надписью "6".

Поиск можно производить не только во всем тексте, но и выделенном фрагменте. Выделите мышью или с помощью клавиатуры какой-нибудь фрагмент на 5 – 10 строк и найдите в нем желаемый текст.

Диалоговое окно поиска и замены текста ReplaceDialog

Программирование кнопки поиска текста служит наглядным примером того, что в ряде случаев написание обработчиков событий отличается большой сложностью. Для того, чтобы уметь программировать события, отличающиеся кодом повышенной сложности, используйте методическое пособие [8] или cправочник среды Delphi [17].

Не станем программировать кнопку поиска и замены текста (кнопка с надписью "7"), поскольку программный код для компонента ReplaceDialog1 отличается еще большей сложностью (желающие могут попробовать свои силы в программировании кнопки самостоятельно). Вместо этого ограни-чимся лишь тем, что на нажатие этой кнопки работающая программа отре-агирует сообщением об отсутствии обработчика события. Создайте обработ-чик кнопки на событие OnClick и введите в него текст

ShowMessage('Эта процедура не запрограммирована ...');

 

Теперь приложение полностью сконструировано и запрограммировано и готово к использованию.

Улучшение внешнего вида кнопок

Должно быть, Вы заметили, что номера на кнопках SpeedButton затрудняют общение с ними, т. к. не дают представления о их назначении по их внешнему виду. Несколько облегчают работу подсказки (hint), однако проблема визуальной идентификации кнопок в целом остается неразрешенной.

Для того чтобы повысить целевую наглядность кнопок, следует исполь-зовать пиктограммы, которые можно наносить на такие кнопки. Это можно сделать с помощью свойства Glyph кнопок.

Рассмотрим пример. Щелкните по кнопке с надписью "1". В окне Object Inspector щелкните по закладке Properties (свойства). Уберите текст из свойства Caption (текст надписи на кнопке). Далее щелкните в строке свойства Glyph (глиф) справа по маленькой кнопке с троеточием. В открывшемся окне щелкните по кнопке "Load …" (загрузить). Далее в еще одном окне найдите папку C:\Program Files\Common Files\Borland Shared\Images\Buttons. В этой папке щелкните по файлу Import.bmp и откройте его. После закрытия окон загрузки пиктограммы на кнопке появится изображение с красной стрелкой, указывающей на лист с текстом. Такое изображение наглядно указывает на то, что кнопка связана с передачей данных в текст и по сравнению с предыдущей надписью на кнопке (цифра 1) способствует быстрому возникновению у пользователя программы ассоциаций о том, что кнопка скорее всего предназначена для открытия файла. Можно использовать вместо Import.bmp рисунок FileOpen.bmp. Аналогично можно подобрать подходящие пиктограммы для других кнопок.

Если Вас не устраивает стандартный набор пиктограмм Delphi, Вы можете создать и поместить на диск собственные пиктограммы. Их можно создать с помощь графических редакторов, например, встроенного в Delphi редактора Image Editor или отдельного мощного редактора изображний Photo Shop (продукт фирмы Adobe), или с помощью специальных программ "вырезания" подходящих изображений с экрана с последующей их доводкой в одном из перечисленных графических редакторов.
