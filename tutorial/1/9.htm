8. ИСПОЛЬЗОВАНИЕ БУФЕРА ОБМЕНА

Пользователи широко известного текстового процессора Microsoft Word часто используют буфер обмена в качестве посредника при выполнении операций пересылки данных. При разработке Delphi-приложений аналогичные операции нами уже применялись при копировании кнопок и других объектов. Среда Delphi розволяет создавать приложения, в которых можно программировать использование буфера обмена. Основными операциями, которые выполняются при использовании буфера обмена, являются операции Copy (копировать), Paste (вставить), Cut (вырезать).

Класс TClipboard

Для применения этих операций используйте класс TClipboard, объявленный в модуле clipbrd (обратите внимание на сокращение). Этот класс не имеет визуального компонента, поэтому имя модуля необходимо добавить в список uses, например:

uses
Windows,Messages,SysUtils,Classes,Graphics,Controls,Forms,
Dialogs,StdCtrls,Buttons,FileCtrl,ComCtrls,ExtCtrls,Clipbrd;

Если в приложении необходимо использовать один буфер обмена, то под него нет нужды создавать специальный объект, поскольку Delphi имеет встроенную переменную Clipboard типа TClipboard, которая создается автоматически. Если необходимо использовать несколько буферов обмена, то следует объявить соответствующие переменные этого типа, создать их с помощью метода Create, а по окончании работы освободить их методом Free.

Тип TClipboard имеет несколько свойств, которые позволяют распознавать тип, размер и формат данных в буфере обмена. В сложных приложениях ими можно целенаправленно управлять. Для стандартных операций обычно достаточно знать только тип данных, с тем, чтобы правильно их использовать и не допустить выполнения некорректных операций во время эксплуатации приложения.

Рассмотрим использование буфера обмена на примере модернизации приложения FolderApp, рассмотренного нами в предыдущей главе.

Приложение для работы с буфером обмена

Запустите Delphi и откройте созданное Вами приложение FolderApp.

Расширим возможности этого приложения, с тем чтобы можно было не только выбирать файлы, но также видеть их содержимое и манипулировать им в зависимости от типа файла. Для упрощения работы ограничимся манипуляциями с текстовыми и графическими файлами.

Если в среде Delphi находится приложение FolderApp, а изображение формы похоже на изображение рис. 6.1, то можно приступать к работе по модернизации приложения.

Модернизация формы fMain

Внесите в список uses модуля main.pas ссылки на два модуля clipbrd, jpeg (последний модуль предназначен для доступа к процедурам обработки изображений в JPEG-формате, эти файлы обычно имеют расширение .jpg). Для этого щелкните по кнопке Toggle Form/Unit. В редактора кода внестие указанные извенения.

Этой же кнопкой верните форму fMain наверх. Нажмите клавишу Shift, и не отпуская ее, последовательно щелкните по всем визуальным компонентам, размещенным на форме. Убедитесь, что все компоненты выделилсь серыми квадратиками. Отпустите клавишу Shift, щелкните в меню Delphi по строке Edit (редактировать), затем в выпавшем окне – по строке Cut (вырезать). Здесь мы использовали буфер обмена Delphi для вырезания выделенных компонентов. После этой операции на форме не должно остаться ни одного компонента (они забраны в буфер обмена).

Немного растяните (на 2 – 3 см) форму по вертикали и горизонтали. Если форма находится слишком низко или слишком смещена вправо, то перед этим щелкните по ее заголовку и отведите ее вверх и влево. Далее подведите курсор мыши к правому нижнему краю формы и, когда курсор мыши примет вид двунаправленной стрелки, расположенной под углом 1350, нажмите левую клавишу мыши и растяните форму до указанных размеров.

Измените значение свойства Caption формы fMain на строку Диски, папки, файлы и буфер обмена (наберите эту строку на клавиатуре).

Щелкните правой клавишей мыши по форме и в выпавшем окне – по строке New Page (новая страница). При этом на компоненте появится новая страница с закладкой TabSheet1.

Измените значение ее свойства Caption на строку символов Диски, папки и файлы (эта надпись должна появиться в заголовке закладки страицы).

Щелкните по этой странице (но не по компоненту PageControl). Чтобы убедиться в правильности выбора обратите внимание на наименование выделенного компонента в окне Object Inspector: под его заголовком должна быть строка TabSheet1: TTabSheet. Далее щелкните в меню Delphi по строке Edit, затем в выпавшем окне – по строке Paste (вставить). При этом на странице TabSheet1 появятся все компоненты, вырезанные ранее в буфер обмена. Если размеры формы слишком велики, уменьшите их с помощью мыши. Внешний вид формы на странице TabSheet1 представлен на рис 8.1.

Снова щелкните (сверху и справа от наименования закладки созданной страницы) правой клавишей мыши по компоненту Page Control (но не по странице TabSheet1). Свойству Caption новой страницы TabSheet2 присвойте значение Текстовые файлы. Она предназначена для манипулирования с текстами и фрагментами текстов текстовых файлов.

Программирование событий

События, относящиеся к элементам страницы TabSheet1, нами были запрограммированы ранее и не требуют доработки. Создадим обработчики событий, которые будут возникать и выполняться, когда пользователь будет щелкать по кнопкам, которые расположены на этих страницах.

Смысл этих событий заключается в том, что выбранный на странице TabSheet1 файл FileName из списка FileListBox, может быть открыт для просмотра и манипуляций его содержимым на странице TabSheet2 или странице TabSheet3. Если выбранный файл является текстовым, то манипуляции с его содержимым можно произвести на странице TabSheet2, если графическим, то – на странице TabSheet3. На каждой из этих страниц размещен соответствующий инструментарий в виде кнопок BitBtn. Щелчок по кнопке будет активизировать событие, предназначенное для выполнения определенных операций.

События для кнопок страницы TabSheet2

Создадим обработчик события от нажатия первой кнопки на странице TabSheet2 (рис. 8.2). Ее имя sbOpenTextFile. Дважды щелкните по кнопке и добавьте в обработчик TfMain.sbOpenTextFileClick следующий код:

Memo1.Lines.LoadFromFile(FileListBox1.FileName);

При выполнении этого события содержимое файла FileName, который был выбран в списке FileListBox1, на странице TabSheet1 будет загружено в редактор Memo1. При этом процедура не проверяет тип загружаемого файла, а представляет его содержимое в этом редакторе в символьной форме.

Верните форму наверх и дважды щелкните по следующей кнопке (ее имя sbTextCopy). Внесите в обработчик TfMain.sbTextCopyClick следующий код:

Memo1.CopyToClipboard;
sbTextInsert.Enabled:= true;

При помощи первого оператора этого кода выделенный в Memo1 текст будет помещен в буфер обмена. При щелчке по кнопке внешне это проявится в том, что соседняя справа кнопка "оживет" (изображение, размещенное на ней станет цветным). Это произойдет потому что во втором операторе свойству Enabled кнопки sbTextInsert будет приcвоено значение True. Это в свою очередь означает, что буфер обмена теперь содержит данные, которыми можно как-либо распорядиться. В нашем приложении мы запрограммируем лишь одну операцию – копирование содержимого буфера обмена в редактор Memo2 (он справа от Memo1).

Верните форму и дважды щелкните по третей кнопке (кнопка с именем sbTextInsert). Внесите в обработчик TfMain.sbTextInsertClick следующую строку:

Memo2.PasteFromClipboard;

Это означает, что при щелчке по кнопке sbTextInsert содержимое буфера обмена следует вставить в редактор Memo2. При этом текст будет вставлен в то место Memo2, где находится клавиатурный курсор.

Отметим, что это событие произойдет лишь тогда, когда эта кнопка уже "ожила"; щелчки по "мертвой" кнопке не дали бы никакого результата.

Запустите приложение. Обратите внимание, что приложение начало работу с той страницы, которая была активна к моменту его запуска. Отсюда следует, что перед запуском приложения нужно активизировать первую страницу (щелкнуть по ее закладке).

Проверьте работу запрограммированных кнопок. Для этого выберите на первой странице какой-нибудь текстовой файл. Таким файлом может быть файл main.pas нашего приложения, который уже размещен на диске в ваших папках, или любой другой файл с расширением .txt (можно также проверить что будет помещено в Меmo1, если выбранный файл не является текстовым).

Щелкните по второй закладке с надписью Текстовые файлы. На этой странице щелкните по первой кнопке. Содержимое выбранного файла появится в Memo1. Выделите в нем какой-нибудь фрагмент текста мышью или клавиатурными клавишами, затем щелкните по второй кнопке. При этом третья кнопка "оживет". Это означает, что выделенный Вами фрагмент текста помещен в буфер обмена. Щелкните по третей кнопке и убедитесь, что скопированный в буфер текст появился в редакторе Memo2. Проделайте в Memo2 какие нибудь манипуляции, например, вставьте нажатием клавиши Enter несколько пустых строк, перенесите строки на другое место и т. д. Теперь поставьте клавиатурный курсор где-нибудь в середине текста и вновь щелкните по кнопке вставки данных из буфера обмена. Убедитесь что вставился тот же самый текст, причем с того места, где был расположен клавиатурный курсор. Очередной щелчок по кнопке приведет к еще одной вставке того же текста и т. д.

Обратите внимание, что при наведении курсора мыши на любую кнопку под ней на короткий промежуток времени появляется подсказка, объясняющие назначение кнопки. Это "срабатывет" свойство Hint (подсказка) и тот факт, что свойство ShowHint (показывать подсказку) установлено в True.

Выберите другой файл и вновь проверьте действие кнопок. Убедитесь также в том, что две кнопки на странице TabSheet2 и все кнопки страницы TabSheet3, за исключением кнопок с надписью Выход, не работают, т. к. еще не запрограммированы.

Закройте приложение кнопкой с надписью Выход. При этом управление будет передано Delphi.

Подолжим программирование кнопок страницы TabSheet2.

Две оставшиеся кнопки sbTextFontPlus и sbTextFontMinus предназначены соответственно для увеличения и уменьшения на 2 единицы размера фонта для текста, размещенного в редакторе Memo1. Запрограммируйте их, внеся в обработчик первой кнопки текст

Memo1.Font.Size:= Memo1.Font.Size + 2;

а второй кнопки – текст

Memo1.Font.Size:= Memo1.Font.Size – 2;

Вновь запустите приложение и убедитесь, что теперь эти кнопки работают и дают ожидаемые результаты.

События для кнопок страницы TabSheet3

Эта страница пердназначена для работы с графическими файлами. К настоящему времени существует множество форматов для графических файлов. К их числу относятся файлы "классических" форматов bmp, jpg, gif, wmf, а также ряд файлов с форматами tif, tiff, dib, rle, psd, png и многие другие.

В нашем приложении мы будем использовать файлы только двух наиболее распространенных форматов – bmp и jpg. Файлы этих форматов имеют расширение .bmp и .jpg, соответственно (например, dot.bmp, arrow.jpg).

Перейдите на страницу TabSheet3. Создайте обработчик события для первой кнопки (ее имя sbOpenGrFile). Она предназначена для вызова изображения, размещенного в файле FileListBox1.FileName, и передачи его в объект Image1 (он слева под надписью Изображение 1).

Текст обработчика приведен ниже

procedure TfMain.sbOpenGrFileClick(Sender: TObject);
var s: String;
begin
With FileListBox1 do
begin
s:= AnsiUpperCase(ExtractFileExt(FileName));
if (s = '.JPG') or (s = '.BMP') then
Image1.Picture.LoadFromFile(FileName);
end;
end;

В процедуре объявлена локальная строковая переменная s, которая используется для определения расширения выделенного файла с помощью оператора

s:= AnsiUpperCase(ExtractFileExt(FileName));

Сначала в нем из полного имени файла выделяется расширение (функция ExtractFileExt), затем оно "поднимается" в верхний регистр (функция AnsiUpperCase). В результате, например, для файла

FileListBox1.FileName = 'c:\Windows\Pictures\ball.jpg'

значением переменой s ,будет s = '.JPG'.

П р и м е ч а н ие. Порядок выполнения этих функций можно было бы поменять метами, однако при этом получился бы худший код. Действительно, сначала обработчику пришлось бы "поднимать" длинную строку в верхний регистр и потом из нее выделять расширение, при противоположном порядлу эти операции будут выполнены быстрее.

Далее проверяется условие, в соответствии с которым, если значением переменной s является строка '.JPG' или (or) '.BMP' (обратите внимание, что строки находятся в верхнем регистре), то (then) объект Picture объекта Image1 следует загрузить (LoadFromFile) из файла, выделенного на странице TabSheet1. Обратите также внимание на то, что использование оператора With позволило уменьшить длину операторов, в которых используется имя файла.

Таким образом, при нажатии первой кнопки не всякий файл будет считан и загружен в изображение, а только тот, который отвечает, вышеприведенному условию.

Вторая кнопка (она имеет имя sbGrCopy) предназначена для копирования загруженного в объект Image1 изображения в буфер обмена.

Дважды щелкните по кнопке и создайте внутренний код процедуры-обработчика. Ее текст состоит из следующих операторов:

procedure TfMain.sbGrCopyClick(Sender: TObject);
begin
if Image1.Picture <> Nil then
begin
Clipboard.Assign(Image1.Picture);
if not sbGrInsert.Enabled then
sbGrInsert.Enabled:= true;
end;
end;

При выполнении процедуры проверяется загружено ли в объект Picture изображение (т. е. не пусто ли оно). Только при положительном ответе будет выполнено два оператора: сначала методом Assign в буфер будет помещено изображение, затем, если кнопка вставки из буфера "мертва" (not sbGrInsert.Enabled, буквально – "не жива"), то она будет "оживлена" (sbGrInsert.Enabled:= true).

Третья кнопка (с имененем sbGrInsert) предназначена для передачи из буфера обмена изображения в объект Image2 (он справа от Image1).

Текст обработчика для щелчка по этой кнопке состоит из следующих операторов:

procedure TfMain.sbGrInsertClick(Sender: TObject);
begin
if Clipboard.HasFormat(CF_PICTURE) then
Image2.Picture.Assign(Clipboard);
end;

Здесь сначала производится проверка формата хранящихся в буфере данных. Если данные имеют тип CF_PICTURE (т. е. тот тип, который можно загрузить в объект Picture), то затем этот объект для изображения Image2 загружается из буфера обмена. В результате на экране в том месте, где помещен объект Image2 появляется изображение, скопированное до этой операции в буфер обмена.

П р и м е ч а н и е. Возникает вопрос, почему подобные меры предосторожности не были предприняты при программировании аналогичных ситуаций для текстовых файлов. Конечно, их можно было бы предпринять (существует формат CF_TEXT), однако для расширения возможностей текстового редактора, это можно не делать, т. к. любой, в том числе графический, файл, считанный как текстовой, будет представлен как последовательность символов, кодирующих это изображение. Совсем по другому обстоит дело, когда совершается попытка представить неграфический файл как изображение. Если не выполнить предварительную проверку типа данных, содержащихся в буфере обмена, то их чтение может привести к ошибке в программе и аварийному ее завершению. Если в программе возникают такие ситуации, которые принято называть исключительными, то они являются признаком несовершенства такой программы.

Запрограммируем последнюю кнопку (с именем sbStretch). Создайте обработчик события и вставьте в него оператор:

Image1.Stretch:= not Image1.Stretch;

Этот оператор меняет значение свойства Stretch логического типа (растягивание) объекта Image1 на противоположное. Если Stretch = False, то в Image1 будет помещено изображение без искажения (из больших изображений будет вырезан фрагмент по размерам объекта Image1, малые будут помещены полностью). Если Stretch = True, то изображение будет растянуто в соответствии с размерами объекта Image1 (при этом, как правило, получается искаженное изображение).

И тот и другой способ использования свойства Stretch имеет недостатки: при первом (False) – большие изображения не вмещаются в Image1, при втором (True) – изображение полностью "в кадре", однако представлено в искаженном виде. Устранить эти недостатки можно дополнительным усовершенствованием кода, который для вмещающихся изображений полагал бы Stretch = False (изображение – в натуральную величину), а для изображений выходящих за рамки Image1 полагал бы Stretch = True и представлял бы рисунок уменьшенным, но с сохранением его пропорций.

Решение этой задачи реализовано в нижеследующей процедуре.

procedure ImageFormat(Var Image: tImage);
var kW,kH: Real;
begin
With Image do
begin
Height:= StartHeight;
Width := StartWidth;
Stretch:=(Picture.Width>Width) or (Picture.Height>Height);
if Stretch then //картинка не вмещается
begin
kW:= Picture.Width/Width; //гориз.пропорция
kH:= Picture.Height/Height; //верт.пропорция
if (kH>kW) then kW:=kH; // выбираем меньшую пропорцию
Height:=Round(Picture.Height/kW); //новая высота
Width:=Round(Picture.Width/kW); // новая ширина
end;
end;
end;

Процедура сама определяет значение Stretch и автоматически форматирует выводимый рисунок. В ней используется две глобальные переменные StartHeight, StartWidth, которые "помнят" первоначальные размеры Image1. Запоминание начальных значений проиводится в обработчике OnCreate для формы в виде операторов

StartHeight:= Image1.Height;

StartWidth := Image1.Width;

Если желаете проверить действие процедуры, положите на страницу TabSheet3 еще одну кнопку и создайте для нее обработчик события, вставив в него оператор

ImageFormat(Image1);

Запустите программу и проверьте действие кнопок страницы TabSheet3. Если Вы не знаете местонахождение графических файлов на диске, обратись за помощью к преподавателю.
