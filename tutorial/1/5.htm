4. ВЫКЛЮЧАТЕЛИ, ПЕРЕКЛЮЧАТЕЛИ И СТРОКИ ВВОДА

Уже знакомые Вам кнопки, метки, панели, а также новые компоненты – переключатели, выключатели и строки ввода – это те визуальные компоненты Delphi, которые наиболее часто применяют при конструировании пользовательского интерфейса (окон управления приложением). Хотя набор этих компонентов относительно невелик, он, тем не менее, позволяет удовлетворить практически любые потребности программиста при использовании как отдельных элементов, так и элементов, которые можно объединять в группы.

В этой главе на примере приложения будет дано объяснение тому как использовать эти компоненты. Кроме того, найдут использование другие компоненты, которые позволяют улучшить интерфейс и украсить его внешний вид.

Новые компоненты

Ниже перечислены новые компоненты, пиктограммы которых расположены на панели палитры визуальных компонентов (рис. 1.1).
 Кнопка настройки SpinButton

азмещена на закладке Samples. Представляет собой две соединенные друг с другом кнопки. Обычно используется для увеличения или уменьшения значения какой-либо целочисленной величины при помощи щелчка мыши по одной из этих кнопок.

ыключатель CheckBox

азмещен на закладке Standard. Компонент состоит из метки (надписи) и небольшого квадратика. Щелкнув по нему, пользователь может включить или выключить какой-нибудь режим, описанный текстом метки. Если необходимо, чтобы исходным было состояние "включено", необходимо установить свойство Checked в True (такое состояние помечено галочкой). Выключатель обычно либо включен, либо выключен, но он может быть и недоступным. Недоступность обозначается серой галочкой. Для создания такого выключателя (с тремя состояниями) следует свойство AllowGrayed установить в True, а затем свойству State присвоить одно из значений cbChecked, cbGrayed, cbUnchecked (значение по умолчанию). Теперь пользователи смогут его выключить, сделать недоступным или включить.
Цвет выключателя можно менять, подобрав для свойства Color нужное значение. Шрифт надписи на выключателе, включая цвет, размер и др., задается свойством Font.


 Группа GroupBox
 Размещена на закладке Standard. Компонент предназначен для логического объединения по некоторому признаку множества переключателей и других объектов в единое целое. Фокус ввода пользователь может перемещать между группами с помощью клавиши <Tab>, а между компонентами текущей группы – с помощью клавиш перемещения курсора.
 Переключатель RadioButton
 Размещен на закладке Standard. Используется редко. В большинстве приложений гораздо удобнее использовать компонент RadioGroup. Если на форме расположено несколько таких компонентов, то пользователи могут выбирать только один из них. Если щелчком выбрать один из них, то в его окошке появится черный кружок, при этом в окошках остальных компонентов этого кружка не будет. Таким образом, компонент RadioButton можно использовать для визуального выбора требуемого варианта из некоторого множества доступных вариантов. В отличие от CheckBox компонент RadioButton реагирует не только на одиночный, но и на двойной щелчок. Таким образом, можно выбирать, а при необходимости и воспроизводить события в зависимости от способа выбора компонента.


 Группа переключателей RadioGroup
 Компонент размещен на закладке Standard. Представляет собой группу переключателей. Для его создания достаточно положить компонент на подложку (например, форму или панель), указать текст каждого переключаемого параметра в свойстве Items и запрограммировать событие, соответствующее щелчку по выбираемому переключателю. Все сказанное о наборе одиночных переключателей RadioButton применимо к этому компоненту.


 Строка ввода Edit
 Размещена на закладке Standard. Предназначена для ввода строки символов с клавиатуры.


 Строка ввода SpinEdit
 Размещена на закладке Standard. Компонент, в котором соединены строка ввода и кнопка настройки.


 Фаска Bevel
 Размещена на закладке Additional. Простейший визуальный компонент, который выглядит как прозрачный прямоугольный выступ или углубление. Обычно на него кладут другие компоненты (чаще кнопки) и используют для декоративных целей. Изобретательные программисты применяют Bevel для создания разделительных буртиков, канавок и линий при разбиении окна на функциональные зоны.


 Часто прозрачная Bevel оказывается поверх компонентов, которые должны, наоборот, лежать ней, поэтому их нельзя, например, выделить (при щелчке по компоненту выделяется фаска). Фаску можно подложить под компоненты следующим приемом. Щелкните по фаске правой клавишей мыши. В выпавшем контекстном меню щелкните по строке Send to Back (отправить вниз). Фаска уйдет под компоненты, с которыми теперь можно работать при помощи мыши.

 Проектирование простейшей базы данных
 Учетные данные об абитуриенте будем хранить в структуре, имеющей тип:
 Type
TAbit = Record
Fio : String[40]; // Фамилия
Year : Word; // Год рождения
Pol : byte; // Пол: 0 – мужской, 1- женский
Hostel: boolean;// Нуждаемость в общежитии:
// True – да, False – нет
End;


 Базу данных об абитуриентах представим в виде файла, состоящего из записей одинакового размера. Дадим ему имя 'abit.rcs'. Файл будем размещать в той же папке, в которой расположен проект Abit.dpr.


 Тип записи TAbit содержит четыре поля. Строковое поле Fio предназначено для хранения фамилии абитуриента. Максимальная длина имени равна 40. Размер памяти, необходимой для хранения поля Fio, с учетом нулевого байта строки составляет 41 байт. Поле Year предназначено для хранения года рождения, размер – 2 байта. Поле Pol будет хранить пол абитуриента. Его значением может быть одно из чисел 0 (женский) или 1 (мужской). Поле Hostel предназначено для хранения данных о нуждаемости абитуриента в общежитии. Это поле имеет булевский тип и может иметь одно из значений: True – нуждается в общежитии, False – нет. Размер последних двух полей – 1 байт. Таким образом общий объем памяти, необходимой для хранения данных об одном абитуриенте составляет 45 байт. Для сравнения объем памяти на дискете составляет приблизительно 1,440,000 байт. Следовательно, на одной дискете можно сохранить файл, содержаший сведения о 32,000 абитуриентах в виде записей типа TAbit.


П р и м е ч а н и е. Профессиональные базы данных устроены сложнее и обычно учитывают большее количество сведений. Поэтому на той же дискете можно сохранить такие данные о существенно меньшем количестве записей.
 Чтобы вставить тип записи TAbit и описание переменнойAbit этого типа необходимо перейти в редактор кода. Для этогов щелкните на панели инструментов Delphi по кнопке Toggle Form/Unit. В появившемся тексте модуля внесите с клавиатуры вышеприведенный текст типа между словами end и var, а также описание переменной Abit: TAbit – после оператора fMain: TfMain как показано в листинге модуля (см. листинг. 4.1).
Сразу вслед за этим вставьте два оператора


 F: File of TAbit;
FileAbit: String;


 Переменная F является файловой переменной, представляющей файл с типом TAbit. Вторая переменная FileAbit предназначена для хранения имени этого файла.


 Программирование событий

Сначала определим имя файла, в котором будут храниться записи учетных данных об абитуриентах. Имя должно быть определено до того, как пользователь получит возможность работать с приложением. Наиболее удобным средством может быть событие, которое происходит при активизации формы fMain. Щелкните по форме. Убедитесь, что в Object Inspector появились данные формы. Щелкните по закладке Events, затем дважды – по значению OnActivate. При этом будет создана заготовка процедуры для обработки этого события. Внесите в тело процедуры оператор FileAbit := 'abit.rcs'. Текст процедуры должен иметь вид


 procedure TfMain.FormActivate(Sender: TObject);
begin
FileAbit:= 'abit.rcs';
end;


 Теперь после запуска приложения во время активизации формы переменная FileAbit получит присвоенное ей значение.
П р и м е ч а н и е. Имя файла можно было бы определить еще проще, задав его перед описанием типа записи TAbit как строковую константу с помощью оператора
Const FileAbit = 'abit.rcs';
 Использованный прием применен здесь для демонстрации возможностей среды.
 Далее запрограммируем кнопку bAddToEndOfFile (на ней надпись "Добавить в конец файла"), которая предназначена для записи учетных данных об абитуриенте, сведения о котором пользователь внес в зоне ввода этих данных. Для этого двойным щелчком по этой кнопке создайте событие OnClick (на щелчок) кнопки и внесите в обработчик события текст так, чтобы процедура приняла вид:


procedure TfMain.bAddToEndOfFileClick(Sender: TObject);
var Pos: Integer;
begin
Case Trim(Abit.Fio)<>'' of
false: ShowMessage('Отсутствует фамилия');
true:
begin
AssignFile(F,FileAbit);
Pos:=0;
Case FileExists(FileAbit) of
true : begin Reset(F); Pos:=FileSize(F); end;
false: Rewrite(F); end; {Case}
Seek(F,Pos);
BlockWrite(F,Abit,1);
CloseFile(F);
end;
end; {Case}
end;


 В этой процедуре описана локальная переменная Pos (ее область действия ограничена пределами этой процедуры), которая предназначена для определения будущего места записи Abit в файле. В теле процедуры содержится два вложенных Case-оператора. Селекторы этих оперторов имеют логический тип. Первый Case-оператор предназначен для выяснения готовности записи в файл. Если фамилия непуста (Trim(Abit.Fio)<>'', т. е. при удалении лидирующих и завершающих пробелов из поля Fio записи Abit с помощью функции Trim образуется непустая запись), то принимается, что такую запись можно поместить в файл, иначе будет выдано модальное окно с сообщением ShowMessage об отсутствии фамилии и процедура закончит работу. Если этот селектор примет истинное значение (True), то будет выполнен оператор связывания файла с файловой переменной F (AssignFile) и обнуление переменной Pos, затем – внутренний Case-оператор, в котором сначала процедура выясняет существует ли файл, в который следует производить запись (FileExists). Если файл не существует, то далее он инициализируется и открывается процедурой Rewrite (при этом местоположение первой записи уже определено как нулевое). Если файл к моменту записи уже существует, то он открывается процедурой Reset, а местоположение записи определяется как количество уже существующих в нем записей (Pos:=FileSize(F)). Дествительно, если в файле 5 записей, то последняя запись находится на месте 4 (первая – на месте 0), и новую запись следует записать на место 5. Далее в открытом одним из этих способов файле, производится перемещение файлового указателя на запись с номером Pos (процедурой Seek) и затем – запись данных об абитуриенте в файл (процедурой BlockWrite). После записи данных оператор CloseFile закрывает файл.


Теперь запрограммируем кнопку bDelFromPos, которая предназначена для удаления записи с заданной позиции файла. Щелкните по ней дважды и создайте обработчик события. Текст процедуры-обработчика имеет вид


 procedure TfMain.bDelFromPosClick(Sender: TObject);
var
k,i,Pos: Integer;
begin
AssignFile(F,FileAbit);
Reset(F);
k:=FileSize(F);
if (k>0) then begin
Pos:=SePos.Value;
if (Pos>=0) and (Pos<k) then
if MessageDlg('Удалить запись ?',mtConfirmation,
[mbYes, mbNo], 0) = mrYes then
begin
for i:= Pos+1 to k-1 do
begin
Seek(F,i);
BlockRead(F,Abit,1);
Seek(F,i-1);
BlockWrite(F,Abit,1);
end;
Seek(F,k-1);
Truncate(F);
end;
end;
CloseFile(F);
end;


 В этой процедуре сначала производится открытие файла и определение количества записей в нем. Если в файле есть записи (k>0) и если позиция, с которой предлагается удалить запись, в файле задана правильно ((Pos>=0) and (Pos<k)), то начинается процесс удаления. Этот процесс сводится к последовательному перемещению записей по следующей схеме: с позиции Pos+1 читается запись (BlockRead) и записывается на на место Pos (BlockWrite), с позиции Pos+2 читается – на место Pos+1 записывается и т. д., пока есть записи. В результате все записи, начиная с Pos+1 будут смещены на одну позицию к началу файла. При этом две последних записи окажутся одинаковы. Чтобы отсечь последнюю запись, указатель устанавливается на нее и файл отсекается до этой записи оператором Truncate. После этого файл закрывается оператором CloseFile.

Программирование этой процедуры может служить наглядным примером трудностей с которыми сталкивается программист при программировании даже простейших операций, каковой является удаление одной лишь записи из одного файла. В профессиональной практике эта и многие другие процедуры, как правило, уже запрограммированы и отлажены и программисту остается лишь воспользоваться готовым программным продуктом. Вместе с тем можно привести немало примеров, когда даже самый совершенный программный продукт не может удовлетворить конкретные запросы программиста. В этом случае программист создает собственный аппарат обработки.
 Наконец, запрограммируем последнюю кнопку – bMemo (с надписью Показать записи. В работающей программе щелчок по этой кнопке вызовет процедуру формирования списка введенных в файл записей и вывода этого списка в редактор Memo с именем Me.


 Щелкните по кнопке дважды и создайте обработчик события, текст которого приведен ниже


 procedure TfMain.bMemoClick(Sender: TObject);
var
k,i: Integer;
s,q,w: String;
begin
Me.Clear;
if FileExists(FileAbit) then
begin
AssignFile(F,FileAbit);
Reset(F);
k:=FileSize(F);
if (k>0) then
begin
q:='нуждается';
for i:=1 to k do
With Abit do
begin
BlockRead(F,Abit,1);
s:=IntToStr(i)+' '+Fio+' '+IntToStr(Year)+' ';
if Hostel then w:='' else w:='не';
s:=s+' '+w+' '+q;
if (Pol=1) then w:='мужской' else w:='женский';
s:=s+' '+w;
Me.Lines.Add(s);
end; {With,for}
end;
CloseFile(F);
end;
end;


 В процедуре объявлено две целочисленных переменных k, i и три строковых переменных s, q, w.
Сначала редактор Me очищается оператором Me.Clear. Затем, если файл существует, то он открывается операторами AssignFile и Reset. Далее при помощи функции FileSize переменная k получает значение, равное количеству записей в файле AbitFile. Если записи в файле есть (k>0), то переменной q приваевается значение 'нуждается'. Это значение будет использовано при формировании строк, выводимых в редактор Me.


Наполнение редактора производится в цикле, который начинается оператором for. Переменная i является счетчиком и будет в цикле последовательно принимать значение от 1 до k. На каждом шаге цикла оператором BlockRead считывается из файла в переменную Abit очередная запись.


 После чтения производится формирование строки s для последующей передачи ее в редактор. Сначала с помощью функции IntToStr целочисленное значение переменной i преобразуется в строковое значение (например, при i=1 будет '1', при 2 – '2' и т. д.). Затем к нему справа добавляется пробел, который используется как разделитель. К этому результату добавляется значение поля Fio (фамилия) записи Abit, еще один пробел и преобразованный в строку год рождения. Теперь строка частично сформирована и содержит порядковый номер абитуриента в списке, его фамилию и год рождения. Осталось добавить сведения о нуждаемости в общежитии и пол. Сначала формируется нуждаемость в общежитии. Поскольку в базе данных в целях экономии памяти нуждаемость характеризуется значением поля Hostel логического типа, то для удобства восприятия его нужно предварительно преобразовать в строковый тип. Это делается в два приема: если Hostel истинно, то строковой переменной w присваевается пустая строка '', иначе строка 'не '. Затем к частично сформированной строке s добавляется w и q, т. е. для нуждающегося добавится подстрока 'нуждается', иначе – 'не нуждается'. Аналогично формируется подстрока, характеризующая пол абитуриента, с той лишь разницей, что приходится преобразовывать целый тип в строковый. Полностью сформированная строка s далее добавляется в редактор Me оператором Me.Lines.Add(s).


 По окончании прохода по всем записям файла редактор будет наполнен соответствующими строками, а файл закрыт оператором CloseFile.
